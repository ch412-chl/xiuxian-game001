# 数值策划

## 目录职责
本目录只存放当前版本正式生效的数值真源。

## 当前核心主表

- `神通配置总表.csv`：神通名称、伤害、CD、回复效果与解锁真源
- `成就配置总表.csv`：成就目标、奖励与排序真源
- `妖物图鉴总表.csv`：图鉴档案、解锁与展示真源
- `广告配置总表.csv`：广告触发、频次限制与广告位占位真源
- `道具配置总表.csv`：丹药、符箓、装备、材料与突破道具真源
- `装备掉落配置总表.csv`：装备掉落关卡与权重真源
- `破境与渡劫配置总表.csv`：突破丹消耗、成功率、冷却与渡劫表现真源
- `历练事件配置总表.csv`：历练事件、文本类型与收益真源
- `轻肉鸽神通战斗配置总表.csv`

## 读取时机
出现以下任务时优先读取本目录：
- 调整数值、奖励、门槛、掉落或解锁条件
- 接入正式配置替换代码中的硬编码
- 判断运行时数值是否偏离正式主表

## 阅读顺序
1. 先读 `关卡修为经验与境界升级表.csv`
2. 若任务涉及人物等级外显或关卡顺序推进，补读 `人物等级境界映射总表.csv` 与 `轻肉鸽关卡映射总表.csv`
3. 再读 `敌人配置总表.csv`
4. 若任务属于轻肉鸽重构，再读对应 `轻肉鸽*总表.csv`
5. 若任务属于轻肉鸽第三轮数值校准或需要确认正式战斗目标，先读 `../分析报告/轻肉鸽重构/第三轮数值校准执行结论.md`
6. 若实现轻肉鸽升级弹层、build 成长、Buff 或地图节奏，优先补读：
   - `../实现规范/局内等级与升级候选池实现级方案.md`
   - `../实现规范/战斗 Buff 与技能升级链实现级方案.md`
   - `../实现规范/战斗地图与模式参数实现级方案.md`
7. 若实现神通 / 成就 / 图鉴 / 广告 / 道具 / 装备 / 事件，则继续读取对应主表
8. 若要理解这些表如何接入运行时，转到 `../实现规范/轻肉鸽真源桥接与中间主表规范.md`、`../实现规范/轻肉鸽战斗边界状态机与写回规则.md` 与 `../实现规范/广告系统实现级方案.md`

## 使用约束
- 不把旧分析表、旧归档表当作当前主表
- 不在未核对主表前直接改代码中的硬编码数值
- 若发现实现规范与主表在数值字段、配置字段、掉落字段上冲突，以主表为准
- 若冲突发生在展示逻辑、状态机或交互结构上，以总 PRD 与实现规范为准
