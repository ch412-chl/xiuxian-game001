# 本地免费出图方案

更新时间：2026-03-18 09:42:00

## 1. 目标
在不依赖 OpenAI API 计费的前提下，为当前项目建立可持续的本地免费出图路径，并与现有资源映射、提示词和自动化文档体系衔接。

## 2. 当前机器判断
当前设备：
- 芯片：Apple M2 Pro
- GPU：19 核
- 内存：32 GB
- 图形能力：Metal 支持

结论：
- 适合本地生成界面资源、战斗背景、妖物立绘
- 适合中小批量自动化出图
- 不适合把超大模型、多并发高分辨率生产作为默认路线

## 3. 推荐路线
### 路线 A：Draw Things
适用场景：
- 最快开始
- 先验证风格
- 适合 Mac 本地单机出图

优点：
- Apple Silicon 兼容友好
- 安装门槛低
- 不依赖 Python 环境

缺点：
- 自动化和批量化弱于 ComfyUI
- 更适合作为前期验证与单机出图工具

### 路线 B：ComfyUI
适用场景：
- 项目长期自动化
- 批量出图
- 后续资源流水线接入

优点：
- 工作流清晰
- 易于和映射、批次、接入流程结合
- 长期更适合本项目的自动化目标

缺点：
- 安装比 Draw Things 复杂
- 仍需额外下载模型文件

## 4. 当前建议
当前推荐采用双阶段路线：
1. 先用 Draw Things 快速验证首批风格
2. 再用已搭好的 ComfyUI 环境承接长期自动化批量出图

原因：
- 当前项目最需要先看到“第一批真图”来验证风格和接入
- ComfyUI 更适合作为第二阶段的长期自动化中枢

## 5. 与当前自动化体系的衔接
无论使用 Draw Things 还是 ComfyUI，统一遵守以下真源：
- `assets/资源映射/妖物资源映射.json`
- `assets/资源映射/战斗背景映射.json`
- `assets/资源映射/界面资源映射.json`

- `product/自动化体系/资源流程/界面资源提示词总表.csv`
