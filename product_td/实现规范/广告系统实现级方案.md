# 广告系统实现级方案

## 1. 适用范围
本文档用于定义当前版本广告系统的正式实现边界、页面入口、状态机、频次限制、失败兜底与数据真源。

当前版本广告只允许作为“挽回损失”或“缩短等待”的补充工具，不允许反客为主成为核心循环本体。

## 2. 当前版本广告定位
当前版本只允许 3 个激励视频广告位：
1. 突破冷却跳过
2. 历练广告复活
3. 历练广告撤离

当前版本明确不做：
- Banner 广告
- 插屏广告
- 结算页强制广告
- 任意弹窗式广告打断

## 3. 单一真源
广告行为与广告位配置的唯一正式真源：
- `product/数值策划/广告配置总表.csv`

实现线程不得把广告位 id、频次限制、触发页面、奖励文案散落写死在多个页面文件中。

## 4. 广告位清单

### 4.1 突破冷却跳过
- 触发页面：人物页
- 触发条件：突破失败且存在冷却
- 展示文案：`观看广告立即跳过冷却`
- 成功结果：当前突破冷却立即清零
- 失败结果：保留原冷却，不额外补偿
- 每日上限：以 `广告配置总表.csv` 为准

### 4.2 历练广告复活
- 触发页面：轻肉鸽战场页 `revive_offer` / 复活确认层
- 触发条件：本次历练战斗失败、当前主状态属于 `wave / boss`、且本局未使用广告复活
- 展示文案：`观看广告原地复活`
- 成功结果：恢复到死亡前主状态并继续战斗
- 失败结果：停留在复活确认层
- 单局上限：每次历练最多 1 次

### 4.3 历练广告撤离
- 触发页面：轻肉鸽战场页 `retreat_confirm` / 撤离确认层
- 触发条件：当前主状态属于 `wave`，无传送符，且点撤离后满足广告资格
- 展示文案：`观看广告保留战利品撤离`
- 成功结果：进入 `settle_preview_retreat -> settle_commit`，带回当前可结算 runBag 战利品
- 失败结果：停留在撤离确认层
- 单局上限：每次历练最多 1 次

## 5. 交互状态约束

### 5.1 突破冷却广告
人物页仅在以下条件同时满足时显示广告入口：
- 当前处于突破冷却状态
- 对应广告位启用
- 当日广告次数未耗尽

不满足时：
- 不显示按钮
- 不显示占位文案

### 5.2 广告复活
复活确认层仅在以下条件同时满足时显示：
- 当前主状态死亡
- 本局尚未使用广告复活
- 对应广告位启用

广告复活成功后：
- 退出复活确认层
- 回到死亡前主状态
- 不允许再次展示本局广告复活入口

广告复活失败后：
- 停留在复活确认层
- 不自动进入失败结算页
- 不自动判定撤离

### 5.3 广告撤离
撤离确认层仅在“无传送符且点撤离”后显示广告撤离。

广告撤离成功后：
- 进入 `settle_preview_retreat`
- 玩家确认后进入 `settle_commit`
- 仅在 `settle_commit` 写回正式库存

广告撤离失败后：
- 停留在撤离确认层
- 不直接吞掉撤离入口

## 6. 数据字段要求
`广告配置总表.csv` 最少包含：
- 广告位 ID
- 广告类型
- 业务场景
- 触发页面
- 触发条件
- 展示文案
- 奖励类型
- 奖励值
- 每日上限
- 单局上限
- 失败兜底
- 开发广告位 ID
- 正式广告位 ID
- 启用状态

## 7. 上线前必填项
上线前必须补齐：
- 微信正式广告位 ID
- 开发环境广告位 ID
- 各广告位日限次
- 广告失败回调的统一埋点事件名

若 `开发广告位ID` 或 `正式广告位ID` 仍为 `TODO_*`，视为不可提审。

## 8. 微信开发者工具验收
必须通过：
1. 突破失败后出现冷却广告入口
2. 广告入口在冷却结束后自动隐藏
3. 复活确认层仅出现 `广告复活 / 认命离开`
4. 本局广告复活成功后不再显示第二次广告复活
5. 无传送符撤离时出现广告撤离选项
6. 广告撤离成功后，runBag 只在 `settle_commit` 写入 `inventory`
7. 广告失败时页面停留在原状态，不得直接吞掉入口

## 9. 禁止事项
1. 禁止在当前版本加入 Banner 或插屏广告
2. 禁止把广告入口常驻展示在非触发状态页面
3. 禁止把广告奖励直接写死在页面逻辑中而不读配置表
4. 禁止广告成功后绕过正常状态写入链路
5. 禁止继续把广告复活挂在失败结算预览页
