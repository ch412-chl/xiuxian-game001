# 战斗 Buff 与技能升级链实现级方案

## 1. 文档定位
本文档定义轻肉鸽战斗中另外两层必须正式拆开的真源：
1. Buff 层
2. 技能升级链层

目的只有一个：
**不要再把所有成长都硬塞进神通主表或强化主表。**

## 2. 正式结论
当前版本需要新增两张正式主表：
1. `轻肉鸽Buff配置总表.csv`
2. `轻肉鸽技能升级链总表.csv`

并与现有两张表形成四层关系：
1. `轻肉鸽神通战斗配置总表.csv`
2. `轻肉鸽技能升级链总表.csv`
3. `轻肉鸽Buff配置总表.csv`
4. `轻肉鸽局内强化配置总表.csv`

## 3. 为什么必须拆
如果不拆，后续实现会出现两种坏结果：
1. 所有效果都写成“神通某字段 +1”
2. 所有强化都变成一次性的数值 patch

这会导致：
- build 看起来很多，实际上人格很弱
- 很难做多段效果、持续效果、连锁效果、叠层效果
- 后面扩内容时每个神通都得单独特判

## 4. 技能升级链层
### 4.1 职责
`轻肉鸽技能升级链总表.csv` 负责定义：
- 一个神通从基础态到终局态的连续成长
- 每个阶段具体增强什么
- 哪些升级是该技能独享的，不适合抽象成通用 Buff

### 4.2 适合放进升级链的效果
- 飞剑数 +1
- 冰锥分裂
- 火弹爆炸半径扩大
- 天雷连锁数增加
- 剑气穿透数增加

### 4.3 不适合放进升级链的效果
- 全局冷却缩减
- 全局拾取范围扩大
- 全局护盾上限增加
- 全局伤害增幅

这些应进入 Buff 层。

### 4.4 技能升级链表最少字段
- `skillUpgradeId`
- `skillId`
- `upgradeLevel`
- `upgradeName`
- `upgradeDesc`
- `changeType`
- `valueA`
- `valueB`
- `valueC`
- `visualTag`
- `isCapstone`

## 5. Buff 层
### 5.1 职责
`轻肉鸽Buff配置总表.csv` 负责定义所有可复用的全局或半全局增益。

### 5.2 正式 Buff 分类
1. 输出类
2. 冷却类
3. 范围类
4. 弹道类
5. 控场类
6. 生存类
7. 资源类

### 5.3 Buff 适用对象
Buff 必须明确作用范围：
- `global`
- `school`
- `skill`
- `projectile`
- `player`

### 5.4 Buff 最少字段
- `buffId`
- `buffName`
- `buffType`
- `targetScope`
- `targetRef`
- `stackRule`
- `maxStack`
- `durationType`
- `durationValue`
- `effectType`
- `valueA`
- `valueB`
- `valueC`
- `displayPriority`

## 6. 强化如何桥接到升级链与 Buff
### 6.1 当前正式规则
`轻肉鸽局内强化配置总表.csv` 中每条升级项，只能落到以下两类目标之一：
1. `skillUpgradeId`
2. `buffId`

不允许再出现第三种隐式目标。

### 6.2 升级项与目标关系
- 神通升阶类：优先指向 `skillUpgradeId`
- 被动道法类：优先指向 `buffId`
- 特殊混合类：允许同时写 `skillUpgradeId` 与 `buffId`，但必须在文档中明确说明

## 7. 展示规则
### 7.1 升级弹层
升级卡面必须能让玩家一眼区分：
- 这是在升某个神通
- 还是在获得全局 Buff

推荐标签：
- `神通升阶`
- `被动道法`
- `流派联动`
- `终局强化`

### 7.2 战斗内反馈
Buff 若会改变：
- 子弹/术法数量
- 范围
- 冷却
- 护盾
- 连锁  
则战斗内必须能通过明显反馈体现，不允许只有后台数值变化。

## 8. 与外部参考的对齐结论
外部参考项目显示，成熟战斗不会把：
- 升级候选
- 子技能效果
- Buff
- 技能表现  
混成一张表。

因此我们当前必须接受这条正式结论：
**轻肉鸽神通战斗配置主表只负责“技能本体”，不再承担全部成长定义。**

## 9. 微信运行态验收
必须验证：
1. 同一神通升级后，手感是否明确变化
2. Buff 与技能升级是否能叠出不同 build 人格
3. 高层 build 是否不是单纯数值膨胀，而是作用方式改变
4. 升级卡面是否能清楚区分“升技能”与“拿 Buff”

## 10. 禁止事项
- 禁止把所有成长都塞回 `轻肉鸽神通战斗配置总表.csv`
- 禁止让 `轻肉鸽局内强化配置总表.csv` 直接承担 Buff 数值定义
- 禁止出现无法回链到 `buffId` 或 `skillUpgradeId` 的升级项
- 禁止把全局被动写成“隐藏效果”不进主表
