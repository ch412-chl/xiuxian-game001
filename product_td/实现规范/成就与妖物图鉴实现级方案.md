# 成就与妖物图鉴实现级方案

## 1. 文档目的
本文档定义当前版本的：
- 成就页
- 妖物图鉴页
- 洞府到成就 / 图鉴的入口关系
- 成就奖励查看与领取
- 妖物图鉴解锁与详情查看

## 2. 设计结论
- `成就页 = 长期目标反馈页`
- `妖物图鉴页 = 敌人档案与收集反馈页`
- 两页都属于 `洞府页` 的次级功能页
- 战场内不展示成就 / 图鉴详情
- 结算页不展示成就 / 图鉴详情
- 若本局新增解锁，只在洞府对应入口显示红点

## 3. 页面关系
洞府页 -> 成就页
洞府页 -> 妖物图鉴页
成就页 / 图鉴页 -> 洞府页

## 4. 成就页规范
### 4.1 页面职责
1. 展示长期目标分类
2. 展示当前成就进度
3. 提供奖励查看与领取

### 4.2 页面结构
1. 顶部返回与标题区
2. 成就分类摘要区
3. 成就列表区
4. 奖励详情弹层

### 4.3 固定规则
- 返回路径固定为洞府页
- 成就分类固定为：`历练 / 境界 / 收集`
- 列表只允许真滚动或分页
- 奖励领取后直接写入纳戒
- 战场内不弹成就完成提示
- 结算页不展开成就详情
- 若本局解锁新成就，只在洞府 `成就` 入口显示红点

## 5. 妖物图鉴页规范
### 5.1 页面职责
1. 展示已解锁妖物档案
2. 补充敌人名称、归属关卡与技能认知
3. 提供收集完成度反馈

### 5.2 页面结构
1. 顶部返回与标题区
2. 图鉴完成度区
3. 妖物列表区
4. 妖物详情弹层

### 5.3 固定规则
- 返回路径固定为洞府页
- 首次遭遇妖物时即解锁图鉴
- 不要求首次击杀才解锁
- 详情只显示名称、关卡归属、类型标签、技能摘要
- 禁止恢复旧气息与旧属性外显
- 战场内不弹图鉴详情
- 结算页不展开图鉴详情
- 若本局解锁新图鉴，只在洞府 `妖物图鉴` 入口显示红点

## 6. 洞府入口规则
- 洞府页必须固定提供 `成就` 与 `图鉴` 入口
- 洞府摘要只显示：
  - 成就：已领取数量 / 可领取数量
  - 图鉴：已解锁数量 / 总数量
- 若存在本局新增：
  - `成就` 入口显示红点
  - `图鉴` 入口显示红点

## 7. 数据与存档规则
### 7.1 成就
- 最少包含：`achievementProgress`、`achievementClaimed`
- 奖励领取后立即写入纳戒

### 7.2 图鉴
- 最少包含：`monsterDexUnlocked[]`
- 遭遇时解锁后立即写入本地存档

## 8. 微信开发者工具验收
### 8.1 成就页
1. 从洞府进入成就页
2. 左上返回能回洞府
3. 列表滚动自然或分页切换正常
4. 可领取成就点击后弹层出现
5. 领取后状态切为 `已领取`
6. 奖励进入纳戒
7. 本局新增成就时，只在洞府入口显示红点

### 8.2 妖物图鉴页
1. 从洞府进入图鉴页
2. 左上返回能回洞府
3. 已解锁数量统计正确
4. 遭遇新妖物后图鉴数量增加
5. 点击已解锁妖物可看详情
6. 详情中不出现旧外显属性与旧气息
7. 本局新增图鉴时，只在洞府入口显示红点

## 9. 明确禁止事项
1. 禁止继续保留成就页伪滚动
2. 禁止继续保留图鉴页伪滚动
3. 禁止把成就 / 图鉴入口放回人物页
4. 禁止在战场内和结算页展开成就 / 图鉴详情
5. 禁止把成就奖励先放入储物袋
