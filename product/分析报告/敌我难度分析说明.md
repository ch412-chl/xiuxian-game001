# 玩家成长与敌人难度分析

版本：v0.9.10
日期：2026-03-18

## 口径
- 玩家基础属性按当前代码默认值：攻击 100、防御 50、速度 80。
- 玩家境界成长按当前 `js/data.js` 与 `js/gameState.js` 的正式公式计算。
- 战斗按当前代码逻辑做确定性近似：不计随机波动、闪避与暴击随机，仅保留当前 CD / 技能轮转 / 回血效果。
- 敌人数据来源于 `product/敌人配置/敌人配置总表.csv`。

## 主要结论
- 当前玩家成长里，`totalAtk` 和 `totalDef` 会增长，但在正式战斗公式里几乎没有完整接入伤害与减伤，因此面板成长感弱于实际需求。
- 玩家 `totalSpd` 当前几乎固定为 80，意味着角色行动节奏主要不随境界成长，只随装备变化。
- 同阶段普通怪和小 Boss 整体可打，但随着境界提升，大 Boss 会逐步进入“耗时更长、容错更低”的区间。
- 若只装 1 个神通，后续大量关卡会偏慢；进入第三个大境界后，按推荐 3 神通装配时，主线难度更接近“合理偏紧”。

## 难度判断标准
- 轻松：战斗时间短，且剩余气血较高。
- 合理：可以稳定取胜，且战斗时长与损耗都在可接受区间。
- 吃力：能赢，但耗时偏长或剩余气血偏低。
- 困难：在当前口径下容易失败。

## 建议
- 优先把 `totalAtk` 接入玩家正式伤害公式。
- 优先把 `totalDef` 接入正式承伤公式，否则防御成长只会体现在气血上限。
- 从第三个大境界开始，建议稳定形成 3 神通轮转；前两境界允许以 1~2 神通过渡。
- 后续若要调难度，优先调 Boss 行动 CD 与 Boss 基础伤害，不要只堆气血。
