# 修仙塔防式自动施法肉鸽需求文档（PRD）

版本：v0.9.66
日期：2026-03-19
最近更新：2026-03-29 16:10:00
文档状态：正式产品文档
适用对象：产品、策划、UI、程序、测试

## 1. 产品概述
- **产品定义**：一款修仙题材微信小游戏，战斗骨架对标《向僵尸开炮》，采用玩家固定底部、自动施法、20 波敌潮推进、局内高频三选一成长的塔防式肉鸽玩法。
- **核心目标**：让玩家在单局内感受到明显的敌潮压力、技能成型和成长爽感，并将单局收益反馈到长期修仙成长。
- **目标人群**：喜欢修仙题材、波次战斗、轻肉鸽成长的移动端玩家。
- **产品范围**：人物页、历练页、技能页、单局战场页、三选一强化、结算页、洞府页、成就页、妖物图鉴页、广告补救、破境成长。

## 2. 产品亮点
- **战斗骨架明确**：玩家固定在底部，不移动，自动攻击范围内敌人，唯一高频决策是升级时三选一卡牌。
- **题材统一**：技能、敌潮、图鉴、破境都用修仙语义表达，不做枪械换皮。
- **成长闭环完整**：局内 build 成型，局外境界成长、纳戒库存、洞府进度持续反馈。

## 3. 核心功能
### 3.1 功能清单
- **人物成长**：展示当前人物等级、修仙境界映射、修为经验、突破条件与主推目标。
- **历练选关**：承接主线推进入口，按通关顺序解锁关卡并直接进入单局战斗。
- **技能页**：展示当前已解锁技能与基础说明。
- **单局战斗**：承载 20 波敌潮、20 级成长、20 次强化的自动施法战斗。
- **单局结算**：结算经验与战利品，并反馈到长期成长。
- **洞府支撑系统**：承接阶段目标、成就、图鉴与长期反馈。
- **破境成长**：在满足条件后完成境界跃迁。
- **广告补救**：提供失败复活、撤离兜底与冷却跳过。

### 3.2 功能说明
- **人物成长**：让玩家看懂自己当前等级对应的修仙阶段，以及下一步该做什么。
- **历练选关**：让玩家按明确关卡编号顺序快速推进主线。
- **技能页**：让玩家在局外看懂自己已解锁技能，不承担装配和排序。
- **单局战斗**：提供当前版本最核心的爽感与成长体验。
- **单局结算**：把单局结果转化为长期成长反馈。
- **洞府支撑系统**：承接长期目标与收集反馈，不反客为主。
- **破境成长**：形成长期阶段跃迁。
- **广告补救**：提供有限兜底，不改变主循环本质。

## 4. 用户流程
- **核心用户流程**：玩家在人物页确认当前成长状态与目标，进入历练页选择主推关卡，必要时进入技能页查看已解锁技能，然后直接进入单局战场；战场内玩家固定在底部自动攻击敌潮，击杀敌人获得局内经验并进行三选一卡牌强化；单局结束后进入结算页，收益反馈到纳戒与长期成长，再回到人物页或洞府页继续推进。
- **关键节点**：
  1. 选定当前已解锁的下一关
  2. 查看已解锁技能
  3. 局内升级三选一
  4. 精英怪波次与首领终局
  5. 战利品写回与结算反馈
  6. 达成条件后的破境
- **可能卡点**：
  1. 玩家不知道当前该推进哪一关
  2. 玩家不知道已解锁技能各自有什么作用
  3. 局内成长感不足，build 不成型
  4. 结算后不清楚收益变化
  5. 达到后期却不清楚为什么不能破境
- **产品要求**：
  1. 人物页和历练页必须明确给出当前已解锁的下一关
  2. 技能页必须清楚展示已解锁技能，不承担装配职责
  3. 战场页必须持续提供波次压力和升级反馈
  4. 结算页必须清楚展示得失与下一步建议
  5. 破境条件必须外显，不允许让玩家靠猜

## 5. 页面与功能说明
### 5.1 人物页
- **页面职责**：展示当前成长状态，并给出下一步行动引导。
- **入口**：底部导航、战斗结算后返回。
- **出口**：历练页、洞府页、破境确认层。
- **核心功能**：
  1. 展示当前等级、境界映射、经验、突破状态
  2. 展示当前主推关卡与主要阻塞
  3. 提供纳戒、破境、洞府入口

### 5.2 历练页
- **页面职责**：承接主线推进入口。
- **入口**：人物页。
- **出口**：单局战场页、技能页、人物页。
- **核心功能**：
  1. 展示按编号排列的关卡清单，如 `1.野狼谷外围`
  2. 只允许已通关关卡的下一关处于可进入状态
  3. 提供进入战斗与进入技能页入口

### 5.3 技能页
- **页面职责**：展示当前已解锁技能与基础说明。
- **入口**：人物页、历练页。
- **出口**：返回来源页。
- **核心功能**：
  1. 展示已解锁技能列表
  2. 展示技能名称、流派、基础效果说明
  3. 作为局外技能认知页

### 5.4 单局战场页
- **页面职责**：承载当前版本的核心战斗体验。
- **入口**：历练页。
- **出口**：三选一强化页、结算页、暂停态。
- **核心功能**：
  1. 玩家固定底部自动攻击范围内敌人
  2. 承载 20 波敌潮推进
  3. 触发局内升级与三选一卡牌强化
  4. 呈现精英怪波次与首领战

### 5.5 三选一强化页
- **页面职责**：提供局内成长选择。
- **入口**：战场升级触发。
- **出口**：返回单局战场页。
- **核心功能**：
  1. 提供 3 张强化卡牌
  2. 从已解锁技能中随机抽取合法强化项，形成 build 成长

### 5.6 结算页
- **页面职责**：收束单局结果并反馈到长期成长。
- **入口**：单局战斗结束。
- **出口**：人物页、洞府页、再次历练。
- **核心功能**：
  1. 展示经验、掉落、损失与写回结果
  2. 给出下一步建议

### 5.7 洞府页
- **页面职责**：承接长期目标、成就与图鉴。
- **入口**：人物页。
- **出口**：成就页、妖物图鉴页、人物页。
- **核心功能**：
  1. 展示阶段目标
  2. 提供成就、图鉴入口
  3. 展示长期进度反馈

## 6. 业务规则
- **触发规则**：
  1. 玩家进入单局后固定进行 20 波战斗
  2. 玩家击杀敌人获得局内经验并触发升级
  3. 玩家每升 1 级固定触发 1 次三选一强化
  4. 第 5 / 10 / 15 波会生成精英怪，第 20 波会生成首领
- **限制规则**：
  1. 玩家在战场内不进行自由移动
  2. 战场内三选一卡牌只能从当前已解锁技能的合法强化项中随机抽取
  3. 洞府只承接支撑系统，不承接挂机生产
  4. 广告只用于失败兜底、撤离兜底和跳过冷却
- **数据规则**：
  1. 修为经验只通过关卡内击败敌人即时获得
  2. 关卡解锁只与通关顺序挂钩，不与人物境界挂钩；通关当前关卡后自动解锁下一关
  3. 历练页关卡命名统一为 `序号.关卡名`
  4. 人物等级按修仙境界映射展示：炼气为 `1~10级`；筑基及以后保持 `初期 / 中期 / 后期`，每个阶段 `10重 = 10级`；当前正式映射总表覆盖 `1~400级`
  5. 波次只影响怪物生成，不影响场内现存怪物去留
  6. 怪物只会因生命归零而消失
  7. 成功撤离 / 通关才允许将本次战利品写回纳戒
  8. 失败 / 认命离开会丢失本次战利品，但经验保留
  9. 妖物图鉴在首次遭遇时解锁，成就奖励领取后直接进入纳戒
  10. 战场内与结算页不展示成就 / 图鉴详情；若有新增解锁，只在对应入口显示红点

## 7. 验收标准
- **功能验收**：
  1. 人物、历练、技能、战场、强化、结算、洞府、成就、图鉴、破境、广告补救全部可用
  2. 单局战斗必须支持 20 波、20 级、20 次强化
- **流程验收**：
  1. 玩家能够从人物页顺畅进入历练、技能、单局战斗、结算并回到长期成长
  2. 长期成长、单局成长、结算反馈三者形成闭环
- **交互验收**：
  1. 战斗体验明确对齐《向僵尸开炮》的塔防式自动攻击骨架
  2. 玩家主决策清晰集中在三选一强化，不出现旧页式战斗主交互
  3. 战场主视觉清楚，非战斗功能不喧宾夺主
