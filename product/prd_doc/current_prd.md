# 修仙版地下城堡需求文档（PRD）

版本：v0.9.64
日期：2026-03-19
最近更新：2026-03-29 10:55:00
文档状态：正式产品文档
适用对象：产品、策划、UI、程序、测试

## 1. 产品概述
- **产品定义**：一款修仙题材的微信小游戏，战斗骨架对标《向僵尸开炮》，采用塔防式自动施法肉鸽玩法。
- **核心目标**：交付一条可运行、可体验、可上线的主链，让玩家在单局战斗中获得明显成长，并把单局收益反馈到长期修仙成长。
- **目标人群**：喜欢修仙题材、轻肉鸽成长和波次战斗的移动端玩家。
- **核心痛点**：传统页式推关战斗缺少持续压迫感与局内成长爽感，玩家很难在单局内感受到 build 成型。
- **产品范围**：人物页、历练页、战前整备页、单局战场页、三选一强化、结算页、洞府页、成就页、妖物图鉴页、广告补救、破境成长。未写入本文档的功能不默认纳入需求。

## 2. 产品亮点
- **亮点 1**：战斗对标《向僵尸开炮》，采用“玩家固定底部、神通自动攻击、敌潮持续压近、升级三选一卡牌强化”的主循环。
- **亮点 2**：修仙题材表达贯穿战斗与成长，技能、敌人、图鉴、破境都统一在修仙语义下呈现。
- **亮点 3**：单局成长与长期成长闭环明确，局内 build 成型、局外境界跃迁、纳戒库存与洞府进度形成持续反馈。

## 3. 核心功能

### 3.1 功能清单
- **人物成长**：展示当前境界、修为经验、突破条件与主推目标。
- **历练与整备**：完成选关、战前整备并进入单局战斗。
- **单局战斗**：承载 20 波敌潮、20 级成长、20 次强化的自动施法战斗。
- **单局结算**：结算经验与战利品，并反馈到长期成长。
- **洞府支撑系统**：承接阶段目标、成就、图鉴与长期反馈。
- **破境成长**：在满足条件后完成境界跃迁。
- **广告补救**：提供有限的失败复活、撤离兜底与冷却跳过。

### 3.2 功能说明
- **人物成长**
  - 功能目标：让玩家看懂自己当前在哪个阶段，以及下一步该做什么。
  - 用户价值：减少迷失感，提升长期成长的明确性。
  - 与其他功能关系：为历练、破境、洞府提供统一入口与目标摘要。
- **历练与整备**
  - 功能目标：承接主线推进入口，并完成战前基础 build 确认。
  - 用户价值：让玩家快速进入当前最该打的关卡，并用低成本完成开局准备。
  - 与其他功能关系：直接连接人物页、单局战场页与结算页。
- **单局战斗**
  - 功能目标：提供当前版本最核心的爽感与成长体验。
  - 用户价值：在一局内获得明显的波次压力、技能成型和强化选择乐趣。
  - 与其他功能关系：读取整备结果，向结算页输出经验与战利品，并影响成就、图鉴、破境。
- **单局结算**
  - 功能目标：把单局结果转化为长期成长反馈。
  - 用户价值：清楚知道这一局得到了什么、失去了什么、下一步该做什么。
  - 与其他功能关系：连接战场页、人物页、洞府页与纳戒库存。
- **洞府支撑系统**
  - 功能目标：承接长期目标反馈和收集反馈。
  - 用户价值：在主战斗之外看到成长、成就和图鉴进度。
  - 与其他功能关系：承接人物成长与战斗结果，不反客为主。
- **破境成长**
  - 功能目标：形成长期阶段跃迁。
  - 用户价值：让长期修炼有明确阶段感和目标感。
  - 与其他功能关系：依赖人物页状态、战斗经验积累与纳戒内资源。
- **广告补救**
  - 功能目标：为失败或受阻玩家提供有限兜底。
  - 用户价值：降低中断和流失风险。
  - 与其他功能关系：与单局战斗、结算和破境系统联动，但不改变主循环本质。

## 4. 用户流程
- **核心用户流程**：玩家在人物页确认当前成长状态与目标，进入历练页选择主推关卡，在战前整备页确认 3 个局外基础神通后进入单局战场；战场内玩家固定在底部自动攻击敌潮，击杀敌人获得局内经验并进行三选一卡牌强化；单局结束后进入结算页，收益反馈到纳戒与长期成长，再回到人物页或洞府页继续推进。
- **关键节点**：
  1. 选定当前主推关卡
  2. 确认战前整备
  3. 局内升级三选一
  4. 精英波与首领终局
  5. 战利品写回与结算反馈
  6. 达成条件后的破境
- **可能卡点**：
  1. 玩家不知道当前该打哪一关
  2. 玩家不知道整备是否合理
  3. 局内成长感不足，build 不成型
  4. 结算后不清楚收益变化
  5. 达到后期却不清楚为什么不能破境
- **产品要求**：
  1. 人物页和历练页必须明确给出当前主推
  2. 战前整备必须低认知成本完成
  3. 战场页必须持续提供波次压力和升级反馈
  4. 结算页必须清楚展示得失与下一步建议
  5. 破境条件必须外显，不允许让玩家靠猜

## 5. 页面与功能说明

### 5.1 人物页
- **页面职责**：展示当前成长状态，并给出下一步行动引导。
- **入口**：底部导航、战斗结算后返回。
- **出口**：历练页、洞府页、破境确认层。
- **核心功能**：
  1. 展示当前境界、经验、突破状态
  2. 展示当前主推关卡与主要阻塞
  3. 提供纳戒、破境、洞府入口
- **基本交互**：
  1. 玩家可直接看到当前阶段目标与下一步动作
  2. 条件满足时可进入破境确认
- **异常场景**：
  1. 条件不足时明确提示缺失项
  2. 无主推关卡时默认回到当前境界第一关

### 5.2 历练页
- **页面职责**：承接主线推进入口。
- **入口**：人物页。
- **出口**：战前整备页、人物页。
- **核心功能**：
  1. 展示当前境界关卡清单
  2. 标记主推关卡
  3. 进入战前整备
- **基本交互**：
  1. 玩家点击关卡后进入整备
  2. 未解锁关卡只展示条件，不允许直接进入
- **异常场景**：
  1. 无已解锁关卡时回落到当前境界起始关卡

### 5.3 战前整备页
- **页面职责**：完成局外基础 build 确认。
- **入口**：历练页。
- **出口**：单局战场页、历练页。
- **核心功能**：
  1. 选择 3 个局外基础神通
  2. 确认出手顺序
  3. 开始单局战斗
- **基本交互**：
  1. 玩家完成配置后开始战斗
  2. 神通少于 3 个时仍允许开局
- **异常场景**：
  1. 未解锁槽位以待解锁态展示，不阻断进入战斗

### 5.4 单局战场页
- **页面职责**：承载当前版本的核心战斗体验。
- **入口**：战前整备页。
- **出口**：三选一强化页、结算页、暂停态。
- **核心功能**：
  1. 玩家固定底部自动攻击范围内敌人
  2. 承载 20 波敌潮推进
  3. 触发局内升级与三选一卡牌强化
  4. 呈现精英波与首领战
- **基本交互**：
  1. 玩家不移动，主决策是升级时选择强化
  2. 战场主视觉始终优先于次级功能
- **异常场景**：
  1. 死亡后进入复活或失败结算
  2. 暂停态下可查看次级信息或退出

### 5.5 三选一强化页
- **页面职责**：提供局内成长选择。
- **入口**：战场升级触发。
- **出口**：返回单局战场页。
- **核心功能**：
  1. 提供 3 张强化卡牌
  2. 形成技能流派与 build 成长
- **基本交互**：
  1. 玩家每升 1 级选择 1 项强化
- **异常场景**：
  1. 候选不足时按保底规则补齐到 3 项

### 5.6 结算页
- **页面职责**：收束单局结果并反馈到长期成长。
- **入口**：单局战斗结束。
- **出口**：人物页、洞府页、再次历练。
- **核心功能**：
  1. 展示经验、掉落、损失与写回结果
  2. 给出下一步建议
- **基本交互**：
  1. 通关、撤离、失败分别展示不同结果
- **异常场景**：
  1. 失败时明确本次战利品损失范围

### 5.7 洞府页
- **页面职责**：承接长期目标、成就与图鉴。
- **入口**：人物页。
- **出口**：成就页、妖物图鉴页、人物页。
- **核心功能**：
  1. 展示阶段目标
  2. 提供成就、图鉴入口
  3. 展示长期进度反馈
- **基本交互**：
  1. 作为轻中枢，不抢主战斗入口
- **异常场景**：
  1. 无新增内容时仍需展示长期目标摘要

## 6. 业务规则
- **触发规则**：
  1. 玩家进入单局后固定进行 20 波战斗
  2. 玩家击杀敌人获得局内经验并触发升级
  3. 玩家每升 1 级固定触发 1 次三选一强化
  4. 第 5 / 10 / 15 波进入精英波，第 20 波进入首领终局
- **限制规则**：
  1. 玩家在战场内不进行自由移动
  2. 玩家局外最多装配 3 个基础神通
  3. 洞府只承接支撑系统，不承接挂机生产
  4. 广告只用于失败兜底、撤离兜底和跳过冷却
- **数据规则**：
  1. 修为经验只通过关卡内击败敌人即时获得
  2. 成功撤离 / 通关才允许将本次战利品写回纳戒
  3. 失败 / 认命离开会丢失本次战利品，但经验保留
  4. 妖物图鉴在首次遭遇时解锁，成就奖励领取后直接进入纳戒
- **异常规则**：
  1. 升级候选不足 3 项时必须补齐到 3 项
  2. 突破条件不足时必须明确展示缺失项
  3. 单局失败后必须明确说明损失与保留内容

## 7. 验收标准
- **功能验收**：
  1. 人物、历练、整备、战场、强化、结算、洞府、成就、图鉴、破境、广告补救全部可用
  2. 单局战斗必须支持 20 波、20 级、20 次强化
- **流程验收**：
  1. 玩家能够从人物页顺畅进入历练、整备、单局战斗、结算并回到长期成长
  2. 长期成长、单局成长、结算反馈三者形成闭环
- **交互验收**：
  1. 战斗体验明确对齐《向僵尸开炮》的塔防式自动攻击骨架
  2. 玩家主决策清晰集中在三选一强化，不出现旧页式战斗主交互
  3. 战场主视觉清楚，非战斗功能不喧宾夺主
