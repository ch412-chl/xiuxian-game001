# 自动化体系

## 目录职责

本目录用于承接与自动化主链相关的制度、运行结果和资源流程文件。

当前固定拆为三层：
- `规则/`：自动化如何运行、如何产出、如何接入
- `报告/`：自动化每轮运行后沉淀的结果文件
- `资源流程/`：后置美术阶段使用的资源提示词、批次说明与资源清单

## 读取时机

出现以下任务时优先读取本目录：
- 需要确认自动化主链如何运行
- 需要查看最近一轮自动化结论或阻塞
- 需要进入资源生成或资源接入阶段

## 阅读顺序

1. 先看 `规则/自动化研发总方案.md`
2. 再看 `报告/本轮自动化报告.md`
3. 若进入后置美术阶段，再看 `资源流程/`

## 真源说明

本目录属于自动化规则与自动化结果说明层。
- 自动化运行原则优先看 `规则/`
- 阶段性结论与阻塞优先看 `报告/`
- 资源提示词、批次与接入说明优先看 `资源流程/`

## 当前有效结构

- `规则/`：自动化长期规则
- `报告/`：自动化阶段报告、进度记录、阻塞清单
- `资源流程/`：资源提示词、批次说明、资源接入说明

## 使用约束

- 不把自动化报告当产品真源或数值真源
- 不在未核对 `规则/` 前直接改自动化口径
- 若当前阶段不做美术，不默认展开 `资源流程/`

## 维护规则

满足以下情况时，增量更新本 README：
- 调整了本目录结构或读取顺序
- 新增了长期有效的自动化规则入口
- 确认了新的长期有效使用边界

以下情况不要更新本 README：
- 只是新增一轮报告
- 只是临时试运行结果
- 只是 `HANDOFF.md` 中的阶段变化
