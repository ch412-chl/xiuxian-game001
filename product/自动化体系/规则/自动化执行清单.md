# 自动化执行清单

## 每轮必须执行
- [ ] 检查 PRD 是否存在缺口或冲突
- [ ] 检查 UI/UE 规格与线框是否冲突
- [ ] 检查资源目录与映射文件是否缺失
- [ ] 检查是否有低风险需求可直接实现
- [ ] 运行基础测试
- [ ] 运行数值回归
- [ ] 运行可玩性检查
- [ ] 更新版本号、最近更新时间与修订日志
- [ ] 输出本轮自动化报告
- [ ] 如无高风险阻塞，则提交 git 并 push

## 发现高风险时
- [ ] 停止自动实现
- [ ] 生成阻塞清单
- [ ] 标记影响模块
- [ ] 给出推荐方案

## 本轮报告最少包含
- [ ] 已自动修复项
- [ ] 阻塞项
- [ ] 重要风险
- [ ] 后续建议
