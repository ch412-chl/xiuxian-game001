# 美术资源自动化规范

## 1. 目标
为妖物立绘、战斗背景和 UI 装饰资源建立可自动识别、可自动映射、可自动接入的统一标准。

## 2. 根目录
项目资源统一放在：
- `assets/`

## 3. 目录结构
- `assets/妖物/完整立绘/`
- `assets/妖物/缩略图/`
- `assets/场景/战斗背景/`
- `assets/界面/图标/`
- `assets/界面/边框/`
- `assets/界面/血条/`
- `assets/资源映射/`

## 4. 命名规则
### 4.1 妖物完整立绘
格式：
`{境界}_{类型}_{敌人ID}_full.png`

示例：
- `炼气_common_qinglang_full.png`
- `炼气_boss_langwang_full.png`

### 4.2 妖物缩略图
格式：
`{境界}_{类型}_{敌人ID}_thumb.png`

### 4.3 战斗背景
格式：
`{章节ID}_{关卡ID}_bg.png`

### 4.4 UI 资源
格式：
`ui_{用途}_{名称}.png`

## 5. 映射文件
### 5.1 妖物资源映射
文件：
- `assets/资源映射/妖物资源映射.json`

职责：
- `enemyId -> full / thumb / realm / type`

### 5.2 战斗背景映射
文件：
- `assets/资源映射/战斗背景映射.json`

职责：
- `stageId -> background`

### 5.3 界面资源映射
文件：
- `assets/资源映射/界面资源映射.json`

职责：
- `uiKey -> filePath`

## 6. 自动接入规则
- 若新增资源文件命名符合规范，自动化可直接补全映射
- 若映射已存在但资源缺失，自动化应报告缺口，不盲目删除映射
- 若资源文件已存在但命名不规范，自动化应优先生成重命名建议
- 若用途不明确，自动化不得自动覆盖既有映射

## 7. 立绘生成要求
- 推荐 PNG
- 推荐透明背景
- 完整立绘与缩略图分开存放
- 同一妖物的完整立绘与缩略图必须共享同一 `enemyId`

## 8. 当前阶段建议
当前优先建设：
1. 妖物完整立绘
2. 妖物缩略图
3. 战斗背景

UI 装饰资源后续逐步补齐。
