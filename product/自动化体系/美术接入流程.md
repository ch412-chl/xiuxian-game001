# 美术接入流程

版本：v0.1  
更新时间：2026-03-18 07:56:51

## 1. 目标
本流程用于规范自动化如何把妖物立绘、战斗背景和界面装饰资源从“需求识别”推进到“代码可引用”的状态。

## 2. 妖物立绘标准流程
1. 读取 `product/数值策划/敌人配置总表.csv`
2. 生成或更新 `assets/资源映射/妖物资源映射.json`
3. 生成 `product/自动化体系/妖物立绘提示词总表.csv`
4. 按提示词产出完整立绘与缩略图
5. 将完整立绘放入 `assets/妖物/完整立绘/`
6. 将缩略图放入 `assets/妖物/缩略图/`
7. 自动检查文件名是否与映射表一致
8. 自动检查图片是否存在、是否可读取、是否满足透明背景要求
9. 更新映射状态并记录到 `product/自动化体系/资源待办.md`
10. 在代码已支持映射读取时，进入“待接入”或“已接入”

## 3. 状态定义
- `待生成`：只有映射骨架，没有实际图片
- `待筛选`：已有多张候选图，但尚未选定正式稿
- `待接入`：正式稿已确认，代码尚未接入
- `已接入`：代码已通过映射表正常读取
- `待替换`：已有图可用，但风格、尺寸或质量不满足当前版本要求

## 4. 文件职责
- `assets/资源映射/妖物资源映射.json`
  - 妖物资源真源
  - 维护敌人 ID、正式文件名、路径、状态和提示词
- `product/自动化体系/妖物立绘提示词总表.csv`
  - 用于 image generation 的批量提示词维护表
- `product/自动化体系/资源待办.md`
  - 用于追踪当前资源缺口和接入阶段

## 5. 接入约束
- 不允许代码直接写死立绘路径
- 不允许生成图片后跳过映射表直接使用
- 不允许在未更新状态的情况下标记为“已接入”
- 若资源与当前 PRD 风格冲突，状态必须标记为“待替换”

## 6. 战斗背景与界面资源
战斗背景和界面资源沿用相同流程：
1. 先建立映射文件
2. 再生成提示词或规格说明
3. 再产出资源
4. 最后接入代码

当前对应映射文件：
- `assets/资源映射/战斗背景映射.json`
- `assets/资源映射/界面资源映射.json`

当前对应提示词表：
- `product/自动化体系/战斗背景提示词总表.csv`
- `product/自动化体系/界面资源提示词总表.csv`
