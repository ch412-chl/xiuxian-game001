# 微信开发者工具运行态验收总表

## 1. 目的
本文档统一当前版本关键主链与支撑系统的运行态 smoke 验收。

## 2. 使用规则
- 所有涉及交互、状态切换、库存写回、破境、广告入口的改动，都必须至少回归一遍本表相关用例。
- 仅通过静态检查、`preview` 成功、代码阅读，不视为通过。
- 若任务属于轻肉鸽主链，还必须同时通过 `轻肉鸽微信运行态验收补充.md`。

## 3. 验收总表
| 编号 | 功能链路 | 前置条件 | 操作步骤 | 预期结果 |
| --- | --- | --- | --- | --- |
| WT-01 | 启动 -> 人物页 | 新存档或已有存档 | 打开小游戏 | 人物页正常展示境界、经验、气血、主推、目标、阻塞、纳戒 / 洞府入口 |
| WT-02 | 人物 -> 轻肉鸽历练选关 | 人物页已加载 | 点击底部历练 | 进入历练选关页，只展示当前关卡列表、主推信息与进入整备入口 |
| WT-03 | 选关 -> 战前整备 | 已在历练选关页 | 点击主推关卡或整备入口 | 进入战前整备页，展示 3 神通顺位 |
| WT-04 | 战前整备替换顺位 | 至少解锁 2 个神通 | 选择顺位 -> 选择神通 | 对应顺位替换成功，摘要同步刷新 |
| WT-05 | 进入单局战场 | 已完成战前整备 | 点击开始历练 | 进入轻肉鸽战场页，不再进入旧探索日志页 |
| WT-06 | 开场准备 -> 20 波敌潮 | 进入单局战场 | 开始战斗并持续推进 | 明确进入 20 波结构，而不是 5 段简化流程 |
| WT-07 | 局内升级 -> 高频强化 | 推进到多次升级 | 连续观察升级节点 | 一局内出现高频三选一，而不是少量固定强化 |
| WT-08 | 精英波出现 | 推进到第 5/10/15 波 | 继续战斗 | 场上出现 3 次精英门槛，区别于普通小怪 |
| WT-09 | 首领第 20 波登场 | 推进到第 19 波结束 | 继续战斗 | 第 20 波先进入 boss_intro，再进入首领战 |
| WT-10 | 通关结算 -> runBag 写入 | 击败首领 | 等待结算 | 通关后先进入预览，再在 `settle_commit` 写回 |
| WT-11 | 失败结算 -> 丢失战利品 | 本局失败且不复活 | 点击认命离开 | 本局 runBag 战利品丢失，不写入纳戒 |
| WT-12 | 广告复活 | 本局未复活且战斗失败 | 点击广告复活 | 返回可继续战斗状态，本局不再显示第二次广告复活 |
| WT-13 | 广告撤离 / 传送撤离 | 单局中已获得战利品 | 点击撤离或使用传送符 | 按正式规则进入撤离确认、预览、写回链 |
| WT-14 | 洞府阶段目标 | 任意进度 | 从人物进入洞府 | 洞府展示当前境界、主推副本、突破条件、成就 / 图鉴摘要 |
| WT-15 | 成就查看与领取 | 有已完成成就 | 从洞府进入成就页 -> 领取 | 奖励写入纳戒 |
| WT-16 | 图鉴解锁与查看 | 首次遭遇某妖物后 | 从洞府进入图鉴页 | 对应妖物已解锁，可查看档案与技能摘要 |
| WT-17 | 可突破态 | 当前大境界后期且条件满足 | 返回人物页 | 出现高亮破境入口 |
| WT-18 | 破境成功链路 | 满足突破条件 | 点击破境 -> 确认 | 播放渡劫演出 -> 切到下一大境界初期 |
| WT-19 | 破境失败冷却链路 | 满足突破条件 | 重复尝试直到失败 | 失败后进入冷却，可显示广告跳冷却 |
| WT-20 | 广告跳冷却 | 处于突破冷却 | 点击广告跳冷却 | 冷却归零，可再次尝试突破 |
| WT-21 | 储物袋回灵丹使用 | 战场内 runBag 存在回灵丹 | 打开储物袋 -> 点击回灵丹 | 当前气血恢复，数量扣减 |
| WT-22 | 储物袋传送符撤离 | 战场内 runBag 存在传送符 | 打开储物袋 -> 点击传送符 | 进入撤离确认与结算链路 |
| WT-23 | 装备掉落成功撤离写回 | 单局内获得装备 | 通关或成功撤离后观察纳戒 | 装备进入 inventory，人物页摘要同步 |
| WT-24 | 纳戒穿戴与人物页同步 | inventory 中存在可穿戴装备 | 进入纳戒页穿戴 / 卸下 | 人物页装备摘要同步刷新 |
| WT-25 | 破境只扣纳戒突破丹 | 满足破境条件且有突破丹 | 点击破境 -> 确认 | 只从 inventory 扣除突破丹 |

## 4. 失败分类口径
1. 页面展示失败
2. 状态机失败
3. 交互失败
4. 数据写回失败

## 5. 禁止事项
1. 禁止只凭 `preview` 成功就判定通过
2. 禁止只做截图，不做状态跳转验证
3. 禁止修改交互后不回归 runBag 写回、失败丢失、突破冷却三条关键链路
