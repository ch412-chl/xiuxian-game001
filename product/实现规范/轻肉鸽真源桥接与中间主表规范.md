# 轻肉鸽真源桥接与中间主表规范

## 1. 文档定位
本文档定义“修仙轻肉鸽自动施法战斗”正式接入运行时前，必须新增或补齐的桥接真源与中间主表。

本文档解决的问题不是“战斗好不好玩”，而是：
- `stageId` 怎么回到主线关卡
- 敌潮、精英、首领、图鉴、掉落之间怎么形成唯一映射
- 后续代码如何读表而不再临时猜关系

## 2. 当前唯一真源关系

### 2.1 轻肉鸽关卡级真源
- `../数值策划/轻肉鸽关卡映射总表.csv`

作用：
- 建立 `stageId` 与主线 42 个关卡的一一映射
- 统一关卡级主题、敌潮模板集、精英模板集、首领机制与掉落组

### 2.2 轻肉鸽战斗级真源
- `../数值策划/轻肉鸽敌潮波次配置总表.csv`
- `../数值策划/轻肉鸽敌群组成总表.csv`
- `../数值策划/轻肉鸽精英配置总表.csv`
- `../数值策划/轻肉鸽首领机制配置总表.csv`
- `../数值策划/轻肉鸽掉落组配置总表.csv`

作用：
- 定义战场中“谁在什么时候出现、以什么形式施压、清掉后给什么”

### 2.3 主线基础真源
- `../数值策划/敌人配置总表.csv`
- `../数值策划/妖物图鉴总表.csv`
- `../数值策划/道具配置总表.csv`

作用：

- 提供图鉴档案
- 提供掉落物引用

## 3. 正式桥接关系

### 3.1 关卡级桥接
`轻肉鸽关卡映射总表.csv` 必须承担以下桥接：
- `stageId -> 大境界 / 阶段 / 关卡名称`
- `stageId -> 主线普通怪引用`
- `stageId -> 主线小Boss引用`
- `stageId -> 主线大Boss引用`
- `stageId -> 图鉴普通怪ID / 图鉴BossID`
- `stageId -> enemyGroupSetId`
- `stageId -> eliteSetId`
- `stageId -> bossMechanicId`
- `stageId -> bossDropGroupId`

### 3.2 敌潮级桥接
`轻肉鸽敌潮波次配置总表.csv` 不再直接承载全部敌人信息，而只负责：
- `stageId -> waveId`
- `waveId -> enemyGroupId`
- `waveId -> eliteId`
- `waveId -> rewardNode`

具体敌群构成由 `轻肉鸽敌群组成总表.csv` 解释。  
具体精英内容由 `轻肉鸽精英配置总表.csv` 解释。

### 3.3 首领级桥接
`轻肉鸽首领机制配置总表.csv` 必须通过以下字段稳定回链：
- `bossId -> stageId`
- `bossId -> bossName`
- `bossId -> rewardDropGroup`

并且 `rewardDropGroup` 必须能在 `轻肉鸽掉落组配置总表.csv` 中找到对应条目。

### 3.4 图鉴级桥接
当前版本图鉴仍以“每大境界 1 普通怪 + 1 Boss”收口，因此：
- 轻肉鸽普通波次不新增独立图鉴条目
- 轻肉鸽精英不直接新增一级图鉴条目，但必须在精英主表中写明 `atlasUnlockRule`
- 首领图鉴仍通过 `图鉴BossID` 回链

## 4. 中间主表职责

### 4.1 轻肉鸽关卡映射总表
当前文件：
- `../数值策划/轻肉鸽关卡映射总表.csv`

职责：
- 统一所有轻肉鸽关卡级入口字段

### 4.2 轻肉鸽敌群组成总表
当前文件：
- `../数值策划/轻肉鸽敌群组成总表.csv`

职责：
- 解释 `enemyGroupId`
- 定义每波敌群的压迫意图、刷怪口、同屏数量边界与伴生单位

### 4.3 轻肉鸽精英配置总表
当前文件：
- `../数值策划/轻肉鸽精英配置总表.csv`

职责：
- 解释 `eliteId`
- 定义精英入场时间、机制、掉落组与图鉴触发规则

### 4.4 轻肉鸽掉落组配置总表
当前文件：
- `../数值策划/轻肉鸽掉落组配置总表.csv`

职责：
- 解释 `dropGroupId`
- 定义精英 / 首领击败后掉落什么、掉到哪里、何时正式写回

## 6. 当前禁止事项
- 禁止继续在运行时用名称模糊匹配 `stageId`
- 禁止继续让 `enemyGroupId / eliteId / dropGroupId` 只做代号不落主表
- 禁止在轻肉鸽代码里直接公式造精英或掉落包
- 禁止让图鉴、掉落、首领机制各自独立维护而不通过桥接真源

## 7. 与旧主线的关系
- 旧 `敌人配置总表.csv` 仍然是主线敌人基础身份真源
- 轻肉鸽中间主表不是替代主线敌人表，而是把主线身份转译成战场级脚本
- 任何后续实现都应该先读：
  1. `轻肉鸽关卡映射总表.csv`
  2. 再读敌潮 / 精英 / 首领 / 掉落四张中间主表
  3. 最后才回到主线敌人表和道具表

## 8. 实现前若仍缺信息
若实现线程发现以下任一项仍无法唯一落地：
- `stageId` 唯一映射
- 图鉴解锁桥接
- 掉落组写回边界
- 精英条目归属

则只能：
1. 在当前文档和任务文档中标出缺失项
2. 给出补充建议
3. 不能直接编码猜测
