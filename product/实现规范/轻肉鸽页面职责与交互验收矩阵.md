# 轻肉鸽页面职责与交互验收矩阵

## 1. 文档定位
本文档只解决一件事：
**把轻肉鸽第二轮“交互与页面”层补到实现线程无需猜测。**

本文档补齐以下实现级信息：
- 页面职责边界
- 状态到页面的唯一映射
- 按钮显隐与禁用规则
- 返回路径与关闭路径
- 异常态与冲突态兜底
- 微信开发者工具运行态验收映射

若与其他文档同时生效，优先级如下：
1. `当前产品需求.md`：决定产品方向
2. `轻肉鸽战斗边界状态机与写回规则.md`：决定状态机与写回
3. 本文档：决定页面职责、按钮显隐、返回路径、异常态与运行态验收口径
4. 其他支撑系统文档：补充具体子系统规则

## 2. 统一命名

### 2.1 页面与层
主页面：
1. 历练选关页
2. 战前整备页
3. 轻肉鸽战场页
4. 三选一强化页
5. 结算预览页

全屏页 / 全屏层：
1. 储物袋页
2. 怪物详情页
3. 撤离确认层
4. 复活确认层
5. 首领登场过场层

### 2.2 状态与页面映射口径
- `retreat_confirm` 对应：撤离确认层
- `revive_offer` 对应：复活确认层
- `boss_intro` 对应：首领登场过场层
- `settle_preview` 对应：结算预览页
- `settle_commit` 不是页面，只是写回执行态
- `leave_to_role` 不是页面，只是离开单局的终态；当前默认落点为 `历练选关页`

### 2.3 结算预览页三种展示变体
以下 3 个名字只用于页面展示分支，不额外新增独立主状态：
1. `settle_preview_win`
2. `settle_preview_retreat`
3. `settle_preview_fail`

### 2.4 冻结总规则
只要战斗内全屏页 / 全屏层被打开，且当前仍处于战斗主状态，则以下内容必须冻结：
- 敌人刷新
- 伤害结算
- 我方与敌方气血变化
- 技能 CD 推进
- 弹体与持续效果计时

例外只有两类：
1. `settle_preview`：本局已结束，不再视为“暂停战斗”
2. `settle_commit`：本局进入写回，不接受玩家操作

## 3. 页面切换总图

### 3.1 主链
人物页 -> 历练选关页 -> 战前整备页 -> `prepare` -> `wave` -> `elite` -> `upgrade` -> `wave` -> `elite` -> `upgrade` -> `boss_intro` -> `boss` -> `settle_preview_*` -> `settle_commit` -> `leave_to_role` -> 历练选关页

### 3.2 战场子链
- `prepare / wave / elite / boss -> 储物袋页 -> 原主状态`
- `wave / elite / boss -> 怪物详情页 -> 原主状态`
- `prepare / wave / elite -> 撤离确认层 -> 原主状态 或 settle_preview_retreat`
- `wave / elite / boss -> 复活确认层 -> 原主状态 或 settle_preview_fail`
- 第二次 `upgrade` 确认后：`boss_intro -> boss`

### 3.3 冲突约束
- 同一时刻最多只允许打开 1 个战斗内子页 / 子层
- 任何子页 / 子层打开时，不允许再打开第二个
- `upgrade`、`boss_intro`、`settle_preview` 打开时，不允许再打开储物袋、怪物详情、撤离确认、复活确认

## 4. 页面职责与按钮矩阵

### 4.1 历练选关页

#### 页面职责
只负责：
- 展示当前大境界 3 关
- 标记当前主推关卡
- 进入战前整备页
- 作为单局结束后的默认回流落点

禁止承担：
- 直接改写神通顺位
- 展示 runBag
- 展示旧探索日志
- 直接写回经验或库存

#### 入口
- 人物页底部导航进入
- 洞府页底部导航进入
- 单局 `leave_to_role` 默认回流进入

#### 必须显示
- 页面标题
- 当前大境界 3 张关卡卡片
- 当前主推标记
- 进入整备入口
- 到人物 / 洞府的底部导航

#### 必须隐藏
- runBag
- 储物袋入口
- 怪物详情入口
- 旧“探索 / 自动推进”双按钮
- 结算按钮

#### 主交互
| 交互 | 结果 |
| --- | --- |
| 点击关卡卡片主体 | 进入对应关卡的战前整备页 |
| 点击进入整备按钮 | 进入对应关卡的战前整备页 |
| 点击人物导航 | 跳转人物页 |
| 点击洞府导航 | 跳转洞府页 |

#### 异常态
- 若关卡真源缺失，卡片允许展示“信息缺失”占位，但必须隐藏进入整备按钮
- 不允许在关卡真源缺失时继续进入战前整备页

### 4.2 战前整备页

#### 页面职责
只负责：
- 展示本局可用 3 神通顺位
- 调整 1 / 2 / 3 号出手顺序
- 确认进入当前关卡战场

禁止承担：
- 直接展示 runBag
- 直接展示怪物详情页
- 直接进行结算或写回
- 展示广告补救入口

#### 入口
- 历练选关页点击关卡卡片或进入整备按钮

#### 必须显示
- 页面标题与当前关卡名
- 3 槽顺位区
- 当前可装配神通区
- 确认进入按钮
- 返回历练按钮

#### 必须隐藏
- runBag
- 储物袋入口
- 怪物详情入口
- 撤离入口
- 结算按钮
- 广告按钮

#### 主交互
| 交互 | 结果 |
| --- | --- |
| 调整神通顺位 | 只更新本局战前配置，不写永久成长 |
| 点击返回历练 | 返回历练选关页 |
| 点击确认进入 | 进入 `prepare`，打开轻肉鸽战场页 |

#### 异常态
- 若当前境界正式解锁神通少于 3 个，未解锁槽位必须显示 `待解锁` 占位，不允许显示空白槽位
- 战前整备页允许以“至少 1 个已解锁神通 + 其余待解锁占位”的状态直接开局
- 只有当 1 号槽没有任何已解锁神通时，才隐藏确认进入按钮
- 若 3 神通战前配置真源缺失，页面必须保留返回按钮并隐藏确认进入按钮
- 不允许在神通配置缺失时直接进入战场

### 4.3 轻肉鸽战场页

#### 页面职责
只负责：
- 承载 `prepare / wave / elite / boss`
- 展示我方战斗信息、敌潮信息、首领信息、runBag 入口
- 打开储物袋页、怪物详情页、撤离确认层
- 产生本局临时经验与 runBag

禁止承担：
- 展示三选一强化
- 展示结算预览
- 处理永久库存
- 展开成就 / 图鉴列表页

#### 主状态按钮矩阵
| 主状态 | 必须显示 | 按钮 / 入口 | 必须隐藏 |
| --- | --- | --- | --- |
| `prepare` | 关卡标题、开场提示、基础神通摘要、我方状态 | `开始`、`储物袋`、`撤离` | `怪物详情`、`强化卡`、`结算按钮` |
| `wave` | 我方气血、敌潮信息、runBag 摘要 | `储物袋`、`怪物详情`、`撤离` | `开始`、`强化卡`、`结算按钮` |
| `elite` | 精英名称、敌潮信息、我方状态 | `储物袋`、`怪物详情`、`撤离` | 旧“遇敌待战”按钮、`强化卡`、`结算按钮` |
| `boss` | 首领名称、首领血条、我方状态、runBag 摘要 | `储物袋`、`怪物详情` | `撤离`、`强化卡`、`结算按钮` |

#### 返回与关闭
- 储物袋页关闭后：回到打开前主状态
- 怪物详情页关闭后：回到打开前主状态
- 撤离确认层取消后：回到打开前主状态
- 不允许从战场页直接返回历练选关页

#### 异常态
- 若进入战场后战场真源缺失，只允许展示错误占位与“返回历练”，不允许继续推进战斗
- 若已打开子层，则战场页本体按钮不响应第二次打开请求

### 4.4 三选一强化页

#### 页面职责
只负责：
- 暂停战斗
- 展示 3 选 1 强化
- 选择后回到战场主状态或进入 `boss_intro`

禁止承担：
- 展示储物袋
- 展示怪物详情
- 展示撤离入口
- 直接显示结算信息

#### 入口
- 对应精英节点清除后进入

#### 必须显示
- 当前强化标题
- 3 张强化卡
- 当前 build 摘要

#### 必须隐藏
- 储物袋
- 怪物详情
- 撤离
- 关闭按钮
- 返回按钮

#### 主交互
| 交互 | 结果 |
| --- | --- |
| 点击任一强化卡 | 记录本局临时强化并退出该页 |
| 第一次强化确认 | 返回 `wave` |
| 第二次强化确认 | 进入 `boss_intro` |

#### 异常态
- 若强化真源在当前强化层级下无法直接提供 3 个合法候选，必须按以下顺序补齐到 3 项：
  1. 先从当前流派兼容池补同流派保底项
  2. 若仍不足，再从通用被动池补“全局增益”保底项
  3. 若仍不足，允许重复出现低阶通用保底项
- 三选一强化页永远必须展示 3 张可点击强化卡，不做 2 选 1、刷新候选或“本次无强化”

### 4.5 结算预览页

#### 页面职责
只负责：
- 展示本局结果
- 等待玩家确认后进入 `settle_commit`
- 作为 win / retreat / fail 三种展示变体的统一容器

禁止承担：
- 直接打开储物袋
- 直接打开怪物详情
- 直接提供广告复活或广告撤离
- 在预览态直接写回 `inventory`

#### 入口
- `boss` 击败首领后进入 `settle_preview_win`
- 撤离确认成功后进入 `settle_preview_retreat`
- 死亡且未复活后进入 `settle_preview_fail`

#### 展示变体矩阵
| 变体 | 必须显示 | 主按钮 | 必须隐藏 |
| --- | --- | --- | --- |
| `settle_preview_win` | 结果标题、经验收益、runBag 战利品、首领击败结果 | `确认结算` | `广告复活`、`广告撤离`、`撤离` |
| `settle_preview_retreat` | 结果标题、经验收益、runBag 战利品、撤离成功说明 | `确认结算` | `广告复活`、`广告撤离`、`撤离` |
| `settle_preview_fail` | 结果标题、保留经验、丢失战利品说明 | `确认离开` | `广告复活`、`广告撤离`、`撤离` |

#### 返回与关闭
- 结算预览页不提供顶部返回
- 结算预览页不响应系统返回到战场
- 点击主按钮后进入 `settle_commit`
- `settle_commit` 完成后统一回到历练选关页

#### 异常态
- `settle_commit` 执行期间，所有按钮必须锁死，不允许重复点击

### 4.6 储物袋页

#### 页面职责
只负责：
- 展示本局 runBag 中可见内容
- 使用允许在单局内使用的道具
- 预览本局装备 / 材料 / 灵石战利品

禁止承担：
- 穿戴 / 卸下装备
- 展示永久纳戒总览
- 直接触发结算预览

#### 入口
- 从 `prepare / wave / elite / boss` 点击 `储物袋`

#### 必须显示
- 页面标题
- 回灵丹（数量 > 0 时）
- 传送符（数量 > 0 时）
- 当前 runBag 灵石
- 当前 runBag 材料
- 当前 runBag 装备掉落预览
- 关闭按钮

#### 必须隐藏
- 0 数量项
- 穿戴 / 卸下按钮
- 突破丹
- 成就 / 图鉴入口
- 结算按钮

#### 主交互
| 交互 | 结果 |
| --- | --- |
| 点击回灵丹 | 立即生效，留在储物袋页并刷新当前气血 |
| 点击传送符 | 关闭储物袋页，进入 `retreat_confirm` |
| 点击装备掉落 | 只展示“通关或成功撤离后写入纳戒”的说明 |
| 点击关闭 | 回到打开前主状态 |

#### 异常态
- 若 runBag 为空，仍允许打开，但只显示“暂无本局物资”与关闭按钮
- 储物袋打开期间不允许叠加怪物详情或撤离确认层

### 4.7 怪物详情页

#### 页面职责
只负责：
- 展示当前主威胁怪物信息
- 补充战场即时认知

禁止承担：
- 展示图鉴全量列表
- 展示 runBag
- 展示结算与掉落写回

#### 入口
- 仅允许从 `wave / elite / boss` 点击 `怪物详情`

#### 必须显示
- 页面标题
- 怪物名称
- 类型标签（普通 / 精英 / 首领）
- 1~2 条机制 / 技能摘要
- 关闭按钮

#### 必须隐藏
- 储物袋
- 撤离
- 结算按钮
- 图鉴列表跳转

#### 返回与关闭
- 关闭后返回打开前主状态
- 打开期间战斗冻结

#### 异常态
- 若当前主威胁怪物真源缺失，页面只保留名称占位、错误提示与关闭按钮

#### 展示口径
- 普通怪：
  - 只显示名称、类型标签、1 条机制摘要
  - 不显示气血、伤害、CD 等数值型字段
- 精英 / 首领：
  - 显示名称、类型标签、当前气血、1~2 条技能摘要
  - 若技能真源存在，可显示技能伤害与 CD
- 怪物详情页的职责是补即时认知，不替代图鉴或完整数值面板

### 4.8 撤离确认层

#### 页面职责
只负责：
- 二次确认是否结束本局并保留当前可结算收益
- 告知保留与丢失项
- 根据资源与广告资格决定显示哪种撤离按钮

禁止承担：
- 展示怪物详情
- 展示强化卡
- 直接写回 `inventory`

#### 入口
- 仅允许从 `prepare / wave / elite` 点击 `撤离`
- 或在储物袋页点击 `传送符` 后进入

#### 必须显示
- 标题
- 当前关卡名
- 当前主状态名
- `将保留什么`
- `将丢失什么`
- `继续历练`

#### 按钮显隐矩阵
| 条件 | 显示 | 隐藏 |
| --- | --- | --- |
| 有传送符 | `使用传送符撤离`、`继续历练` | `广告撤离` |
| 无传送符且广告可用 | `广告撤离`、`继续历练` | `使用传送符撤离` |
| 无传送符且广告不可用 | `继续历练`、撤离不可用提示 | `使用传送符撤离`、`广告撤离` |

#### 返回与关闭
- 点击 `继续历练`：回到打开前主状态
- 点击 `使用传送符撤离`：进入 `settle_preview_retreat`
- 点击 `广告撤离` 成功：进入 `settle_preview_retreat`
- 点击 `广告撤离` 失败：停留在当前层

#### 异常态
- `boss`、`upgrade`、`boss_intro`、`settle_preview` 中调用撤离必须无响应，不允许绕过规则强开本层

### 4.9 复活确认层

#### 页面职责
只负责：
- 在可复活时提供广告复活
- 在不可继续时给出认命离开入口

禁止承担：
- 打开储物袋
- 打开怪物详情
- 直接写回 `inventory`
- 提供广告撤离

#### 入口
- 仅允许在 `wave / elite / boss` 死亡后，且满足广告复活条件时进入

#### 必须显示
- 标题
- 当前死亡前主状态
- 复活收益说明
- `广告复活`
- `认命离开`

#### 必须隐藏
- `关闭`
- `返回`
- `广告撤离`
- `撤离`
- `储物袋`
- `怪物详情`

#### 返回与关闭
- 点击 `广告复活` 成功：进入 `revive_resume`，然后回到死亡前主状态
- 点击 `广告复活` 失败：停留在当前层
- 点击 `认命离开`：进入 `settle_preview_fail`
- 若本局不满足复活条件：跳过本层，直接进入 `settle_preview_fail`

#### 异常态
- 本层打开后，不允许再回到 `prepare` 或旧“遇敌待战”

### 4.10 首领登场过场层

#### 页面职责
只负责：
- 在第二次强化后做首领登场收束
- 展示首领名称与两条机制摘要
- 作为 `boss` 的唯一前置

禁止承担：
- 撤离
- 储物袋
- 怪物详情
- 强化二次选择
- 结算展示

#### 入口
- 第二次强化确认后立即进入

#### 必须显示
- 首领名称
- 两条机制摘要
- 最终阶段提示
- `继续` 按钮

#### 必须隐藏
- `撤离`
- `储物袋`
- `怪物详情`
- `返回`
- `关闭`

#### 返回与关闭
- 点击 `继续`：进入 `boss`
- 不允许手动返回上一状态

#### 异常态
- 进入本层前必须清理残余普通怪
- 本层期间若尝试打开其他子层，必须无响应

## 5. 异常态与门禁总规则

### 5.1 真源缺失态
若页面所需最小真源缺失，只允许：
- 显示标题
- 显示错误或缺失提示
- 保留最小返回 / 关闭能力

禁止：
- 自动跳下一个页面
- 伪造默认奖励
- 伪造默认怪物
- 伪造默认强化项

### 5.2 页面冲突态
若某层已打开：
- 不允许再开第二层
- 不允许通过系统返回回退到已失效状态
- 不允许在 `settle_commit` 期间再次点击主按钮

### 5.3 写回门禁
- 页面层永远不直接写 `inventory`
- 只有 `settle_commit` 能做正式写回
- 失败预览只允许写经验，不允许写 runBag 战利品

## 7. 当前页层口径结论
此前第二轮交互与页面代理遗留的 3 个悬而未决口径，现已正式收口：
1. 三选一强化凑不满 3 项时，必须按“同流派保底 -> 通用被动保底 -> 低阶通用重复保底”补齐到 3 项
2. 战前整备页在正式解锁神通少于 3 个时，未解锁槽位显示 `待解锁` 占位，并允许以当前已解锁神通直接开局
3. 怪物详情页对普通怪不展示数值型字段，只展示名称、类型与机制摘要；精英 / 首领才展示当前气血与技能数值摘要
