# 实现规范

## 目录职责

本目录只存放“实现级方案”与“真源接入规范”。

设计目的：
- 将总 PRD 与实现方案分层
- 避免实现线程继续在 `产品需求/` 中混读总需求与实现细节
- 让接手线程一进入目录就能明确：这里的文档不是背景说明，而是直接约束代码实现的规范

当前目录职责：
- `历练主链实现级方案.md`：定义历练选关、战前整备与探索入口的实现规则
- `探索与战斗实现级方案.md`：定义探索、遇敌待战、战斗、结算与储物袋的实现规则
- `敌人与关卡脚本真源接入规范.md`：定义关卡经验表与敌人配置表如何接入运行时
- `人物页与洞府中枢实现级方案.md`：定义人物页、洞府页、纳戒入口与破境入口的实现规则
- `成就与妖物图鉴实现级方案.md`：定义成就页、妖物图鉴页、奖励领取与解锁反馈的实现规则
- `广告系统实现级方案.md`：定义突破冷却广告、广告复活、广告撤离的实现规则
- `破境与渡劫实现级方案.md`：定义破境入口、突破确认、渡劫演出、失败冷却与广告跳冷却
- `微信开发者工具运行态验收总表.md`：定义当前版本统一 smoke 验收项
- `实现前自检流程.md`：定义实现线程编码前必须执行的信息完整性自检
- `道具与装备轻实现级方案.md`：定义道具使用、储物袋/纳戒、装备掉落与穿戴的当前版本规则
- `历练事件实现级方案.md`：定义探索中的非战斗事件、事件收益与状态切换规则
- `实现优先级与重构顺序.md`：定义当前代码改造的先后顺序、依赖关系与禁止并行项

## 读取时机

出现以下任务时优先读取本目录：
- 调整主链交互或核心页面结构
- 将真源配置接入运行时代码
- 修正代码与 PRD、UI/UE 的落地偏差
- 规划实现顺序、重构顺序或验收项

## 阅读顺序

1. 与当前任务直接相关的实现级方案
2. 对应真源接入规范
3. 验收或自检文件
4. 其他同目录文档按需阅读

## 真源说明

本目录属于实现约束层。
- 若总 PRD 与实现规范发生冲突，以总 PRD 为产品方向真源，以实现规范为当前版本的代码落地约束
- 实现线程不得跳过本目录直接按个人理解修改主链交互

## 维护规则

满足以下情况时，增量更新本 README：
- 新增了正式实现规范文件
- 调整了本目录文件职责或阅读顺序
- 确认了新的长期有效实现约束
- 清理了过期规范口径

以下情况不要更新本 README：
- 只是一次小修复
- 只是会话过程记录
- 只是 `HANDOFF.md` 的阶段变化
