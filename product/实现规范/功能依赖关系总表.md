# 功能依赖关系总表

## 1. 文档定位
本文档只做一件事：
**把当前版本各功能之间的输入、输出、依赖、写回和禁止反向耦合关系写成总表。**

它不是新的 PRD，也不是新的状态机全文。
它是给实现线程、测试线程和接力 AI 快速判断“先做什么、依赖谁、不能跨过谁”的总控表。

## 2. 读取规则
使用顺序：
1. 先读 `当前产品需求.md`
2. 再读 `模块需求导航.md`
3. 再用本表判断依赖顺序
4. 再进入具体实现规范与主表

## 4. 轻肉鸽重构专属依赖顺序
1. 先接 `轻肉鸽关卡映射总表.csv`
2. 再接 `轻肉鸽敌群组成总表.csv / 轻肉鸽精英配置总表.csv / 轻肉鸽首领机制配置总表.csv`
3. 再接 `轻肉鸽敌潮波次配置总表.csv`
4. 再接 `轻肉鸽掉落组配置总表.csv`
5. 再做页面状态机与显隐
6. 最后才启动数值校准

## 5. 最容易实现错的地方
1. 把 `runBag` 当永久库存
2. 在 `settle_preview` 就写回 inventory
3. 把局内强化写进局外神通配置
4. 让图鉴靠击杀解锁，而不是首次遭遇/首领登场
5. 让广告直接改正式数值，而不是做有限补救
6. 用旧探索层数脚本硬套轻肉鸽战场

## 6. 使用要求
若某次实现无法在本表中唯一找到：
- 输入来源
- 输出对象
- 写回位置
- 上游依赖
- 下游影响

则只能：
1. 说明缺失项
2. 给出补充建议
3. 不得直接编码猜测
