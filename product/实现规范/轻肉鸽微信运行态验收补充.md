# 轻肉鸽微信运行态验收补充

## 1. 文档定位
本文档为轻肉鸽战斗主链新增一组独立的微信开发者工具运行态验收项，编号统一为 `WT-LR-*`。

当前口径：
- 旧 `WT-*` 仍保留用于旧主链或共用页面
- 涉及轻肉鸽战斗重构的实现，必须同时通过本文件中的 `WT-LR-*`

## 2. 旧 WT 的处理原则

### 2.1 保留
继续保留：
- 人物页
- 洞府
- 纳戒
- 成就
- 图鉴
- 破境/渡劫
- 共用按钮与返回路径

### 2.2 改写
以下旧验收项若涉及旧探索战斗链路，必须改写为轻肉鸽口径：
- 进入关卡流程
- 战斗态按钮显隐
- runBag / inventory 结算
- 广告复活
- 撤离

### 2.3 废弃
以下口径在轻肉鸽版本中废弃：
- 旧探索页“自动推进 / 探索”双按钮为战斗入口
- 旧“遇敌待战”作为默认战斗前置状态
- 旧“层数推进”作为战斗主进度表达

## 3. 轻肉鸽 WT-LR 用例组

### WT-LR-01 进入关卡即进入战场
- 前置：选择一个轻肉鸽关卡
- 操作：从历练页进入
- 预期：直接进入 `prepare` / 战场页，不出现旧探索日志页

### WT-LR-02 单局主状态闭环
- 前置：任意一关
- 操作：完整打完一局
- 预期：能观察到 `prepare -> wave -> elite -> upgrade_pause -> boss_intro -> boss -> settle_preview/commit`

### WT-LR-03 三选一暂停规则
- 前置：进入强化节点
- 操作：停留 5 秒不选择
- 预期：敌人不继续刷新、伤害不继续结算、气血不变化、CD 不推进

### WT-LR-04 战斗内储物袋暂停
- 前置：在 `wave / elite / boss` 中打开储物袋
- 操作：停留 5 秒后关闭
- 预期：打开期间战斗冻结，关闭后回到原状态

### WT-LR-05 怪物详情暂停
- 前置：在 `wave / elite / boss` 中打开怪物详情
- 操作：停留 5 秒后关闭
- 预期：打开期间战斗冻结，关闭后回到原状态

### WT-LR-06 广告复活恢复原状态
- 前置：在 `wave / elite / boss` 任一主状态死亡，且本局尚未复活
- 操作：观看广告复活
- 预期：回到死亡前主状态，不回旧待战态；runBag 与局内强化保留

### WT-LR-07 认命离开丢失 runBag
- 前置：在失败预览态拥有 runBag 战利品
- 操作：认命离开
- 预期：经验保留，runBag 丢失，不写入 inventory

### WT-LR-08 传送撤离成功写回 inventory
- 前置：在 `wave / elite / boss` 期间拥有传送符与战利品
- 操作：主动撤离
- 预期：经验保留，runBag 在 `settle_commit` 写入 inventory

### WT-LR-09 广告撤离成功写回 inventory
- 前置：无传送符、有广告撤离资格
- 操作：广告撤离
- 预期：经验保留，runBag 在 `settle_commit` 写入 inventory

### WT-LR-10 Boss 过场独立存在
- 前置：推进到首领前
- 操作：观察进入首领阶段
- 预期：出现 `boss_intro` 过场；普通战斗态结束后再进入 `boss`

### WT-LR-11 首领击败结算分两步
- 前置：击败首领
- 操作：观察结算
- 预期：先进入 `settle_preview_win`，再进入 `settle_commit`；不允许首领死亡同帧直接把战利品写入 inventory

### WT-LR-12 局内强化不污染局外整备
- 前置：本局通过强化获得临时神通/被动
- 操作：通关或失败后重新进入新局
- 预期：局内强化全部清空；局外 3 神通整备不被污染

## 4. 微信开发者工具实际操作要求
- 必须以开发者工具里的页面跳转、按钮点击、弹层显示和结算结果为准
- 不允许只看 `preview` 成功或静态构建结果就判定通过
- 每个 WT-LR 必须至少记录：
  - 前置状态
  - 操作步骤
  - 观察结果
  - 是否与文档一致
