# 轻肉鸽微信运行态验收补充

## 1. 文档定位
本文档为轻肉鸽战斗主链新增一组独立的微信开发者工具运行态验收项，编号统一为 `WT-LR-*`。

当前口径：
- 旧 `WT-*` 仍保留用于旧主链或共用页面
- 涉及轻肉鸽战斗重构的实现，必须同时通过本文件中的 `WT-LR-*`
- 第二轮交互与页面验收，以 `轻肉鸽交互与页面总方案.md` 与 `轻肉鸽页面职责与交互验收矩阵.md` 为页面真源

## 2. 旧 WT 的处理原则

### 2.1 保留
继续保留：
- 人物页
- 洞府
- 纳戒
- 成就
- 图鉴
- 破境 / 渡劫
- 共用按钮与返回路径

### 2.2 改写
以下旧验收项若涉及旧探索战斗链路，必须改写为轻肉鸽口径：
- 进入关卡流程
- 战斗态按钮显隐
- runBag / inventory 结算
- 广告复活
- 撤离
- 失败页广告入口

### 2.3 废弃
以下口径在轻肉鸽版本中废弃：
- 旧探索页“自动推进 / 探索”双按钮为战斗入口
- 旧“遇敌待战”作为默认战斗前置状态
- 旧“层数推进”作为战斗主进度表达
- 旧失败预览页同时承载广告复活 / 广告撤离

## 3. 轻肉鸽 WT-LR 用例组

### WT-LR-01 进入关卡即进入战场
- 前置：选择一个轻肉鸽关卡
- 操作：从历练页进入
- 预期：进入战前整备后，可直接进入 `prepare` / 战场页，不出现旧探索日志页

### WT-LR-02 单局主状态闭环
- 前置：任意一关
- 操作：完整打完一局
- 预期：能观察到 `prepare -> wave -> elite -> upgrade -> boss_intro -> boss -> settle_preview -> settle_commit`

### WT-LR-03 三选一暂停规则
- 前置：进入强化节点
- 操作：停留 5 秒不选择
- 预期：敌人不继续刷新、伤害不继续结算、气血不变化、CD 不推进

### WT-LR-04 战斗内储物袋暂停并回原状态
- 前置：在 `prepare / wave / elite / boss` 中打开储物袋
- 操作：停留 5 秒后关闭
- 预期：打开期间战斗冻结，关闭后回到打开前主状态

### WT-LR-05 怪物详情暂停并回原状态
- 前置：在 `wave / elite / boss` 中打开怪物详情
- 操作：停留 5 秒后关闭
- 预期：打开期间战斗冻结，关闭后回到打开前主状态

### WT-LR-06 撤离确认层按钮显隐正确
- 前置：分别准备 3 种场景：有传送符、无传送符但广告可用、两者都不可用
- 操作：在 `prepare / wave / elite` 点击撤离
- 预期：按钮显隐只允许出现当前条件对应的撤离入口；`boss` 中不得出现撤离确认层

### WT-LR-07 广告复活恢复原状态
- 前置：在 `wave / elite / boss` 任一主状态死亡，且本局尚未复活
- 操作：观看广告复活
- 预期：回到死亡前主状态，不回旧待战态；runBag 与局内强化保留

### WT-LR-08 认命离开进入失败结算预览
- 前置：在 `wave / elite / boss` 死亡，并进入复活确认层
- 操作：点击认命离开
- 预期：进入 `settle_preview_fail`；失败预览不再显示广告复活或广告撤离

### WT-LR-09 传送撤离成功写回 inventory
- 前置：在 `prepare / wave / elite` 期间拥有传送符与战利品
- 操作：主动撤离并使用传送符
- 预期：先进入 `settle_preview_retreat`，确认后进入 `settle_commit`；runBag 在 `settle_commit` 写入 inventory

### WT-LR-10 广告撤离成功写回 inventory
- 前置：在 `prepare / wave / elite` 期间无传送符、有广告撤离资格，并已获得战利品
- 操作：主动撤离并完成广告撤离
- 预期：先进入 `settle_preview_retreat`，确认后进入 `settle_commit`；runBag 在 `settle_commit` 写入 inventory

### WT-LR-11 Boss 过场独立存在
- 前置：推进到第二次强化结束
- 操作：观察进入首领阶段
- 预期：出现独立 `boss_intro` 过场；过场期间不允许打开储物袋、怪物详情与撤离；确认继续后再进入 `boss`

### WT-LR-12 首领击败结算分两步
- 前置：击败首领
- 操作：观察结算
- 预期：先进入 `settle_preview_win`，再进入 `settle_commit`；不允许首领死亡同帧直接把战利品写入 inventory

### WT-LR-13 局内强化不污染局外整备
- 前置：本局通过强化获得临时神通 / 被动
- 操作：通关或失败后重新进入新局
- 预期：局内强化全部清空；局外 3 神通整备不被污染

### WT-LR-14 历练选关页只承载选关与整备入口
- 前置：返回历练主链
- 操作：观察历练选关页
- 预期：只展示当前大境界 3 关、主推信息与进入整备入口；不出现 runBag、旧探索按钮与战场按钮

### WT-LR-15 战前整备页返回与开战路径正确
- 前置：已进入战前整备页
- 操作：分别点击返回历练、确认进入
- 预期：返回历练时回到历练选关页；确认进入时进入 `prepare`，不插入旧探索日志页

### WT-LR-16 结算预览页按钮收口正确
- 前置：分别制造 `settle_preview_win`、`settle_preview_retreat`、`settle_preview_fail`
- 操作：观察结算页按钮
- 预期：每种预览都只保留单一确认按钮；不出现储物袋、怪物详情、广告复活、广告撤离、撤离按钮

### WT-LR-17 settle_commit 锁定与回流正确
- 前置：任一结算预览页点击确认
- 操作：连续点击确认按钮并观察回流
- 预期：`settle_commit` 期间按钮锁定不重复触发；写回完成后统一回到历练选关页

## 4. 微信开发者工具实际操作要求
- 必须以开发者工具里的页面跳转、按钮点击、弹层显示和结算结果为准
- 不允许只看 `preview` 成功或静态构建结果就判定通过
- 每个 `WT-LR` 必须至少记录：
  - 前置状态
  - 操作步骤
  - 观察结果
  - 是否与文档一致

## 5. 当前已定口径与剩余上线阻塞
### 5.1 已正式收口
1. 三选一强化若无法直接提供 3 个合法候选，必须按“同流派保底 -> 通用被动保底 -> 低阶通用重复保底”补齐到 3 项
2. 战前整备页若当前境界正式解锁神通少于 3 个，未解锁槽位显示 `待解锁` 占位，并允许以当前已解锁神通直接开局
3. 怪物详情页对普通怪不展示数值型字段，只展示名称、类型与机制摘要；精英 / 首领才展示当前气血与技能数值摘要

### 5.2 仍属上线阻塞
1. 广告真实 `adUnitId` 仍未填充，上线前需重新执行广告链路验收
