# 轻肉鸽交互与页面总方案

## 1. 文档定位
本文档定义“修仙轻肉鸽自动施法战斗”版本下的正式页面集合、页面职责、状态流转、按钮显隐和页面间依赖关系。

本文档解决的问题不是视觉细化，而是：
- 轻肉鸽版本到底有哪些页面
- 每个页面只负责什么
- 各状态下哪些按钮必须显示、哪些必须隐藏
- 页面切换时 runBag、经验、强化、图鉴与广告怎么挂接

## 2. 正式页面集合

### 2.1 主页面
1. 历练选关页
2. 战前整备页
3. 轻肉鸽战场页
4. 三选一强化页
5. 结算预览页

### 2.2 独立子页面 / 全屏层
1. 储物袋页
2. 怪物详情页
3. 撤离确认层
4. 复活确认层
5. 首领登场过场层

## 3. 页面职责

### 3.1 历练选关页
只负责：
- 展示当前大境界三关
- 明确当前主推关卡
- 进入战前整备

禁止承担：
- 直接展开 3 神通详细替换
- 展示 runBag 内容
- 展示旧探索层数日志

### 3.2 战前整备页
只负责：
- 展示本局局外可用 3 神通
- 调整 1 / 2 / 3 号出手顺序
- 确认进入战场

禁止承担：
- 直接结算经验
- 直接展示怪物波次细表

### 3.3 轻肉鸽战场页
只负责：
- prepare / wave / elite / boss / settle_preview 主流程
- 展示敌潮、首领、局内强化前后的即时战场信息
- 驱动本局 runBag 临时写入

禁止承担：
- 持久库存管理
- 成就详情浏览
- 图鉴列表浏览

### 3.4 三选一强化页
只负责：
- 暂停战斗
- 给出 3 选 1 强化
- 选择后回到战场主状态

禁止承担：
- 背包管理
- 长文本剧情说明

### 3.5 结算预览页
只负责：
- 展示本局经验、战利品、首领击败结果、是否通关 / 撤离 / 失败
- 等待玩家确认后进入 settle_commit

禁止承担：
- 在展示阶段直接写回纳戒

## 4. 主状态映射到页面

| 主状态 | 所在页面 | 视觉重点 | 必须显示 | 必须隐藏 |
| --- | --- | --- | --- | --- |
| prepare | 轻肉鸽战场页 | 开局准备与首波提示 | 关卡标题、开始按钮、我方基础神通、主推提示 | 撤离确认层、强化层、结算层 |
| wave | 轻肉鸽战场页 | 敌潮压迫 | 我方气血、敌潮、神通、runBag入口 | 三选一强化层、结算层 |
| elite | 轻肉鸽战场页 | 精英门槛 | 精英名称、敌潮、神通、runBag入口 | 旧“遇敌待战”按钮 |
| upgrade | 三选一强化页 | 构筑选择 | 三个强化项、当前 build 摘要 | 战斗继续推进 |
| boss_intro | 首领登场过场层 | 威压提示 | 首领名称、两条机制摘要、确认继续 | runBag页、强化页 |
| boss | 轻肉鸽战场页 | 首领对峙 | 首领血条、神通、我方血条、runBag入口 | 撤离按钮 |
| settle_preview | 结算预览页 | 本局收束 | 本局经验、本局战利品、结果标题 | 纳戒即时写回提示 |

## 5. 页面切换规则

### 5.1 历练主链
历练选关页 -> 战前整备页 -> 轻肉鸽战场页

### 5.2 战场主链
prepare -> wave -> elite -> upgrade -> wave -> elite -> upgrade -> boss_intro -> boss -> settle_preview

### 5.3 子层打开规则
- 储物袋页：只能从轻肉鸽战场页打开，打开时必须冻结战斗
- 怪物详情页：只能从战场页打开，打开时必须冻结战斗
- 撤离确认层：只允许在 prepare / wave / elite 打开
- 复活确认层：只允许在失败预览前打开

## 6. 按钮显隐规则

### 6.1 历练选关页
显示：
- 关卡卡片
- 当前主推标签
- 进入整备按钮

隐藏：
- runBag
- 旧探索按钮
- 旧自动推进按钮

### 6.2 战前整备页
显示：
- 3 槽顺序区
- 可用神通区
- 确认进入按钮
- 返回历练按钮

隐藏：
- runBag
- 结算按钮
- 广告按钮

### 6.3 战场页
prepare / wave / elite 显示：
- runBag入口
- 怪物详情入口
- 允许撤离时显示撤离入口

upgrade 显示：
- 只显示强化选择卡
- 其他战斗操作全部隐藏

boss_intro 显示：
- 只显示首领摘要与继续按钮

boss 显示：
- runBag入口
- 怪物详情入口
- 禁止显示撤离入口

settle_preview 显示：
- 确认结算按钮
- 失败时的认命离开 / 广告复活 / 广告撤离（按条件）

## 8. 写回依赖
- runBag 只在战场中累积
- 经验可在本局结束时结算保留
- 战利品必须在 settle_commit 时写入 inventory
- 失败认命离开时，经验保留、runBag 掉落丢失

## 9. 与支撑系统的关系
- 成就统计：由战场内击杀、通关、首领击败事件写入
- 图鉴解锁：普通怪首次遭遇、首领首次登场写入
- 广告：仅战场失败复活、战场撤离、破境冷却三类
- 破境：不在战场页触发，只依赖本局结算写回后的局外状态
