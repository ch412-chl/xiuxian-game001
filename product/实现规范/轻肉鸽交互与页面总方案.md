# 轻肉鸽交互与页面总方案

## 1. 文档定位
本文档定义“修仙轻肉鸽自动施法战斗”版本下的正式页面集合、页面职责、状态流转、按钮显隐和页面间依赖关系。

本文档回答的是总问题：
- 轻肉鸽当前正式有哪些页面与层
- 每个页面只负责什么
- 战斗主状态分别落在哪个页面
- 哪些入口允许打开，哪些必须隐藏
- runBag、经验、强化、图鉴与广告挂在哪一层

实现级细节另见：
- `轻肉鸽页面职责与交互验收矩阵.md`

## 2. 正式页面集合

### 2.1 主页面
1. 历练选关页
2. 战前整备页
3. 轻肉鸽战场页
4. 三选一强化页
5. 结算预览页

### 2.2 全屏页 / 全屏层
1. 储物袋页
2. 怪物详情页
3. 撤离确认层
4. 复活确认层
5. 首领登场过场层

## 3. 页面职责总表

| 页面 / 层 | 只负责 | 禁止承担 |
| --- | --- | --- |
| 历练选关页 | 展示当前大境界 3 关、标记主推、进入战前整备 | 展示 runBag、修改战斗态、展示旧探索日志 |
| 战前整备页 | 配置本局 3 神通顺位、确认进入战场 | 直接结算、展示怪物详情、展示广告补救 |
| 轻肉鸽战场页 | 承载 `prepare / wave / elite / boss`，展示即时战况与 runBag 入口 | 展示强化页、展示结算页、处理永久库存 |
| 三选一强化页 | 暂停战斗、提供 3 选 1 强化、确认后返回主流程 | 打开储物袋、怪物详情、撤离 |
| 结算预览页 | 展示 win / retreat / fail 结果，并等待进入 `settle_commit` | 预览态直接写回纳戒、提供复活 / 撤离广告 |
| 储物袋页 | 展示本局 runBag，并在战斗内使用允许使用的道具 | 穿戴装备、展示永久纳戒总览 |
| 怪物详情页 | 展示当前主威胁怪物信息 | 展示图鉴全量列表、展示结算信息 |
| 撤离确认层 | 告知保留 / 丢失项，并确认是否撤离 | 直接写回 `inventory` |
| 复活确认层 | 在可复活时提供广告复活，在不可继续时给认命离开 | 提供广告撤离、打开展示子页 |
| 首领登场过场层 | 清理首领前收束，展示首领名称与两条机制摘要 | 撤离、强化、储物袋、结算 |

## 4. 主状态映射到页面

### 4.1 统一口径
- `retreat_confirm` 对应：撤离确认层
- `revive_offer` 对应：复活确认层
- `boss_intro` 对应：首领登场过场层
- `settle_preview` 对应：结算预览页
- `settle_commit` 不是页面，只是写回执行态
- `leave_to_role` 不是页面，只是离开单局的终态；当前默认回流到历练选关页

### 4.2 主状态映射表
| 主状态 | 所在页面 | 视觉重点 | 必须显示 | 必须隐藏 |
| --- | --- | --- | --- | --- |
| `prepare` | 轻肉鸽战场页 | 开局准备与首波提示 | 关卡标题、开始按钮、基础神通摘要、储物袋入口、撤离入口 | 怪物详情、强化层、结算层 |
| `wave` | 轻肉鸽战场页 | 敌潮压迫 | 我方气血、敌潮信息、储物袋入口、怪物详情入口、撤离入口 | 强化层、结算层 |
| `elite` | 轻肉鸽战场页 | 精英门槛 | 精英名称、敌潮信息、储物袋入口、怪物详情入口、撤离入口 | 旧“遇敌待战”按钮、强化层、结算层 |
| `upgrade` | 三选一强化页 | 构筑选择 | 3 张强化卡、当前 build 摘要 | 储物袋、怪物详情、撤离 |
| `boss_intro` | 首领登场过场层 | 最终阶段压场提示 | 首领名称、两条机制摘要、继续按钮 | runBag 页、怪物详情、撤离 |
| `boss` | 轻肉鸽战场页 | 首领对峙 | 首领血条、我方状态、储物袋入口、怪物详情入口 | 撤离、强化层 |
| `settle_preview` | 结算预览页 | 本局收束 | 结果标题、经验、本局战利品或丢失说明、确认按钮 | 储物袋、怪物详情、广告撤离、战场按钮 |

### 4.3 结算页展示变体
以下 3 个名字只用于展示分支，不额外新增独立主状态：
1. `settle_preview_win`
2. `settle_preview_retreat`
3. `settle_preview_fail`

## 5. 页面切换规则

### 5.1 历练主链
历练选关页 -> 战前整备页 -> 轻肉鸽战场页

### 5.2 战场主链
`prepare -> wave -> elite -> upgrade -> wave -> elite -> upgrade -> boss_intro -> boss -> settle_preview -> settle_commit -> leave_to_role`

### 5.3 子页 / 子层打开规则
- 储物袋页：只能从 `prepare / wave / elite / boss` 打开，打开时必须冻结战斗
- 怪物详情页：只能从 `wave / elite / boss` 打开，打开时必须冻结战斗
- 撤离确认层：只允许在 `prepare / wave / elite` 打开
- 复活确认层：只允许在 `wave / elite / boss` 死亡后、且满足广告复活条件时打开
- 首领登场过场层：只允许在第二次强化确认后打开
- 结算预览页：只允许在 `boss` 击败首领、撤离成功、或死亡后不复活时进入

### 5.4 返回路径唯一口径
- 历练选关页返回：人物页 / 洞府页
- 战前整备页返回：历练选关页
- 储物袋页 / 怪物详情页 / 撤离确认层：关闭后回到打开前主状态
- 三选一强化页：只能通过选择一项退出，不提供关闭与返回
- 复活确认层：只能通过广告复活成功或认命离开退出
- 首领登场过场层：只能通过继续按钮进入 `boss`
- 结算预览页：只能通过确认按钮进入 `settle_commit`
- `settle_commit` 完成后：统一回到历练选关页

## 6. 按钮显隐总规则

### 6.1 历练选关页
显示：
- 关卡卡片
- 当前主推标签
- 进入整备按钮
- 人物 / 洞府导航

隐藏：
- runBag
- 储物袋
- 怪物详情
- 旧探索按钮
- 旧自动推进按钮

### 6.2 战前整备页
显示：
- 3 槽顺位区
- 可用神通区
- 确认进入按钮
- 返回历练按钮
- 未解锁槽位的 `待解锁` 占位

隐藏：
- runBag
- 储物袋
- 怪物详情
- 结算按钮
- 广告按钮

### 6.3 轻肉鸽战场页
`prepare` 显示：
- `开始`
- `储物袋`
- `撤离`

`wave / elite` 显示：
- `储物袋`
- `怪物详情`
- `撤离`

`boss` 显示：
- `储物袋`
- `怪物详情`

轻肉鸽战场页统一隐藏：
- 结算按钮
- 强化卡
- 旧“遇敌待战”按钮组

### 6.4 三选一强化页
显示：
- 3 张强化卡
- build 摘要

补充口径：
- 若当前强化层级下合法候选不足 3 项，必须按“同流派保底 -> 通用被动保底 -> 低阶通用重复保底”补齐到 3 项
- 不允许出现 2 选 1、刷新候选或“本次无强化”

隐藏：
- 储物袋
- 怪物详情
- 撤离
- 返回
- 关闭

### 6.5 首领登场过场层
显示：
- 首领名称
- 两条机制摘要
- `继续`

隐藏：
- 储物袋
- 怪物详情
- 撤离
- 返回
- 关闭

### 6.6 结算预览页
统一显示：
- 结果标题
- 经验结果
- 战利品结果或丢失说明
- 单一确认按钮

统一隐藏：
- 储物袋
- 怪物详情
- 广告复活
- 广告撤离
- 撤离
- 旧战场按钮

### 6.7 怪物详情页
普通怪展示：
- 名称
- 类型标签
- 1 条机制摘要

精英 / 首领展示：
- 名称
- 类型标签
- 当前气血
- 1~2 条技能摘要
- 技能伤害与 CD（真源存在时）

## 8. 写回依赖
- runBag 只在战场中累积
- 经验在本局结束后进入 `settle_commit` 写回人物经验
- 战利品只允许在 `settle_commit` 写入 `inventory`
- 失败认命离开时，经验保留，runBag 战利品丢失
- 广告复活成功时，runBag、局内强化与死亡前主状态保留

## 9. 与支撑系统的关系
- 成就统计：由战场内击杀、通关、首领击败事件写入
- 图鉴解锁：普通怪首次遭遇、首领首次进入 `boss_intro` 时写入
- 广告：仅战场复活、战场撤离、破境冷却 3 类
- 破境：不在战场页触发，只依赖本局结算写回后的局外状态
- 道具：回灵丹与传送符只通过储物袋页进入战场交互
