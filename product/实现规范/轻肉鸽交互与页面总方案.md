# 轻肉鸽交互与页面总方案

## 1. 文档定位
本文档定义修仙轻肉鸽版本的正式页面集合、页面职责、主状态承载关系、按钮显隐与返回路径。

这份文档只回答 4 个问题：
1. 当前局内有哪些页面与层
2. 每个页面只负责什么
3. 20 波主状态落在哪个页面
4. 哪些入口允许打开，哪些必须隐藏

## 2. 正式页面集合
### 2.1 主页面
1. 历练选关页
2. 战前整备页
3. 轻肉鸽战场页
4. 三选一强化页
5. 结算预览页

### 2.2 全屏页 / 全屏层
1. 储物袋页
2. 怪物详情页
3. 撤离确认层
4. 复活确认层
5. 首领登场过场层

## 3. 页面职责总表
| 页面 / 层 | 只负责 | 禁止承担 |
| --- | --- | --- |
| 历练选关页 | 展示当前大境界 3 关、主推信息、进入整备 | 展示 runBag、旧探索日志、战斗态按钮 |
| 战前整备页 | 配置本局 3 神通顺位，确认进入战场 | 结算、广告补救、怪物详情 |
| 轻肉鸽战场页 | 承载 `prepare / wave / elite / boss` 实时战斗 | 展示强化卡、结算页、永久库存 |
| 三选一强化页 | 暂停战斗、提供 3 选 1 强化、确认后返回原战斗链 | 储物袋、撤离、怪物详情 |
| 结算预览页 | 展示通关 / 撤离 / 失败结果，等待进入 `settle_commit` | 直接写回纳戒、继续战斗 |
| 储物袋页 | 展示本局 runBag 与战场内可用道具 | 展示永久纳戒总览 |
| 怪物详情页 | 展示当前主威胁单位信息 | 展示图鉴全量列表 |
| 撤离确认层 | 告知保留 / 丢失项，并确认是否撤离 | 直接写回 `inventory` |
| 复活确认层 | 提供广告复活或认命离开 | 广告撤离、结算写回 |
| 首领登场过场层 | 第 20 波开始前收束普通敌潮并提示首领机制 | 撤离、强化、储物袋 |

## 4. 主状态映射
| 主状态 | 所在页面 | 必须显示 | 必须隐藏 |
| --- | --- | --- | --- |
| `prepare` | 轻肉鸽战场页 | 关卡标题、开始按钮、基础神通摘要、储物袋入口、撤离入口 | 强化页、结算层 |
| `wave` | 轻肉鸽战场页 | 我方气血、波次、等级、经验条、储物袋、怪物详情、撤离 | 强化层、结算层 |
| `elite` | 轻肉鸽战场页 | 精英名称、波次、等级、经验条、储物袋、怪物详情、撤离 | 旧待战按钮、强化层 |
| `upgrade` | 三选一强化页 | 3 张强化卡、当前 build 摘要、当前波次与当前等级 | 储物袋、怪物详情、撤离 |
| `boss_intro` | 首领登场过场层 | 首领名称、两条机制摘要、继续按钮 | 储物袋、怪物详情、撤离 |
| `boss` | 轻肉鸽战场页 | 首领血条、我方状态、波次、等级、经验条、储物袋、怪物详情 | 撤离、强化层 |
| `settle_preview` | 结算预览页 | 结果标题、经验、本局战利品或丢失说明、确认按钮 | 储物袋、怪物详情、广告撤离 |

## 5. 主链唯一口径
### 5.1 历练主链
历练选关页 -> 战前整备页 -> 轻肉鸽战场页

### 5.2 20 波战斗主链
`prepare -> wave(1~4) -> elite(5) -> wave(6~9) -> elite(10) -> wave(11~14) -> elite(15) -> wave(16~19) -> boss_intro -> boss(20) -> settle_preview -> settle_commit -> leave_to_role`

### 5.3 局内升级插入规则
- 玩家在任意 `wave / elite / boss` 中升级时，必须插入 `upgrade`
- `upgrade` 与固定波次无关，与精英节点也无绑定关系
- 每升 1 级固定触发 1 次 3 选 1
- 每局等级上限 20，理论最多 20 次强化选择

## 6. 页面切换规则
- 储物袋页：只能从 `prepare / wave / elite / boss` 打开，打开时必须冻结战斗
- 怪物详情页：只能从 `wave / elite / boss` 打开，打开时必须冻结战斗
- 撤离确认层：只允许在 `prepare / wave / elite` 打开
- 复活确认层：只允许在 `wave / elite / boss` 死亡后、且满足广告复活条件时打开
- 首领登场过场层：只允许在第 20 波开始前打开，不允许再绑定“第几次强化后”
- 结算预览页：只允许在通关、主动撤离成功、或死亡后不复活时进入

## 7. 按钮显隐总规则
### 7.1 历练选关页
显示：关卡卡片、主推标签、进入整备按钮、人物 / 洞府导航
隐藏：runBag、储物袋、怪物详情、旧探索按钮、旧自动推进按钮

### 7.2 战前整备页
显示：3 槽顺位区、可用神通区、确认进入、返回历练、未解锁槽位 `待解锁` 占位
隐藏：runBag、怪物详情、结算、广告按钮

### 7.3 轻肉鸽战场页
- `prepare`：显示 `开始 / 储物袋 / 撤离`
- `wave / elite`：显示 `储物袋 / 怪物详情 / 撤离`
- `boss`：显示 `储物袋 / 怪物详情`
- 统一隐藏：结算按钮、强化卡、旧“遇敌待战”按钮组

### 7.4 三选一强化页
显示：3 张强化卡、当前 build 摘要
补充：若合法候选不足 3 项，必须按“同流派保底 -> 通用被动保底 -> 低阶通用重复保底”补齐到 3 项。
隐藏：储物袋、怪物详情、撤离、返回、关闭

### 7.5 怪物详情页
- 普通怪：名称、类型标签、1 条机制摘要
- 精英 / 首领：名称、类型标签、当前气血、1~2 条技能摘要、技能伤害与 CD（真源存在时）

## 8. 数据依赖
- 历练选关页：`轻肉鸽关卡映射总表.csv`
- 战前整备页：`神通配置总表.csv`、`轻肉鸽神通战斗配置总表.csv`
- 战场相关：`轻肉鸽敌潮波次配置总表.csv`、`轻肉鸽敌群组成总表.csv`、`轻肉鸽精英配置总表.csv`、`轻肉鸽首领机制配置总表.csv`、`轻肉鸽掉落组配置总表.csv`、`轻肉鸽局内等级配置总表.csv`

## 9. 写回依赖
- runBag 只在战场中累积
- 经验在本局结束后进入 `settle_commit` 写回人物经验
- 战利品只允许在 `settle_commit` 写入 `inventory`
- 失败认命离开时，经验保留，runBag 战利品丢失
- 广告复活成功时，runBag、局内强化与死亡前主状态保留

## 10. 当前禁止事项
1. 禁止继续把旧探索日志页当轻肉鸽战场前置页
2. 禁止把 runBag 展开在战场主视图下半区
3. 禁止在 `boss` 状态显示撤离入口
4. 禁止把三选一强化做成侧边小弹框
5. 禁止把旧“遇敌待战”按钮组直接套到轻肉鸽战场页
6. 禁止把广告复活 / 广告撤离继续挂在失败预览页上
