# 轻肉鸽战场UI信息组织对齐方案

## 1. 文档定位
本文档只解决一个问题：
**轻肉鸽战场页在信息组织、交互主次、战斗中断与反馈上，如何对齐《向僵尸开炮》的塔防式肉鸽体验。**

它不定义：
- 波次数值
- 强化内容
- 掉落强度

## 2. 第一性原则
真正对齐的关键不是“长得像”，而是战斗主语要对：
1. **玩家固定在底部防线，不参与移动**
2. **敌潮推进永远是主视觉**
3. **升级三选一是唯一高频主交互**
4. **次级功能全部让位给战场和升级**

## 3. 正式信息层级
### 3.1 第一层：战场核心层
必须占据绝对主视觉：
- 底部玩家与防线主体
- 中上部敌潮推进
- 自动施法弹道
- 首领进场与威压

### 3.2 第二层：战斗状态层
始终服务于“我守得怎么样”：
- 当前波次
- 当前等级
- 当前经验条
- 当前气血
- 首领血条（出现时）

### 3.3 第三层：次级系统层
只保留极弱入口：
- 暂停
- 仅在暂停层内再进入储物袋 / 详情 / 退出

### 3.4 第四层：中断层
只在必要时覆盖战场：
- 升级三选一
- 首领登场提示
- 复活确认
- 结算预览

## 4. 正式布局
### 4.1 常规战斗态
推荐结构：
- 顶部细 HUD：波次 + 等级 + 经验 + 气血
- 中上：敌潮推进主区域
- 底部中央：玩家 / 防线主体
- 角落：暂停按钮

### 4.2 首领战
新增：
- 顶部首领血条
- 首领机制短标签

同时隐藏：
- 撤离
- 常规次级入口扩展态

### 4.3 升级弹层
升级弹层必须居中覆盖战场，且：
- 一次只出现 3 张卡
- 背后战场暂停
- 关闭后直接回战场，不经过中间页

## 5. 交互主次
### 5.1 战场中玩家的第一决策
不是走位，也不是点底部按钮，而是：
- 观察敌潮推进
- 判断当前 build 是否成型
- 升级时选哪张卡

### 5.2 第二决策
- 首领压近时看清机制与压力来源

### 5.3 第三决策
- 暂停后是否查看储物袋 / 怪物详情 / 退出

所以：
**底部功能条必须废除，暂停菜单取代常驻三按钮。**

## 6. 按钮与弹层规则
### 6.1 暂停按钮
- 只能在战场角落以弱存在出现
- 点击后冻结战斗并打开暂停菜单

### 6.2 储物袋
- 不常驻
- 只能从暂停菜单进入
- 打开后保持冻结战斗

### 6.3 怪物详情
- 不常驻
- 只能从暂停菜单或首领登场提示里进入
- 首领详情优先于普通怪

### 6.4 撤离 / 退出
- 不做战场主按钮
- 只能放进暂停菜单
- 首领战阶段默认禁用

### 6.5 升级弹层
- 不允许右侧抽屉
- 不允许底部半屏
- 必须是全局主中断层

## 7. 与《向僵尸开炮》的对齐要求
### 7.1 必须对齐的地方
- 玩家固定在底部
- 战场始终是主视觉
- 升级是高频插入式中断
- 敌潮压力读感强
- build 成长通过战斗内反馈而不是结算页体现

### 7.2 修仙化改写
不能出现枪械 UI 语义：
- 不出现枪械弹匣感
- 不出现现代军武边框
- 不出现枪械数值词汇

必须转换成修仙语义：
- 波次像妖潮推进
- 强化像悟道 / 升法
- 首领像妖王压场

## 8. 页面显隐硬规则
### 8.1 常规波次
显示：
- 波次
- 等级
- 经验
- 气血
- 暂停按钮

隐藏：
- 旧页式待战按钮
- 储物袋常驻入口
- 怪物详情常驻入口
- 撤离常驻按钮
- 神通整备入口

### 8.2 升级态
显示：
- 3 张强化卡
- 当前等级
- 当前波次
- 当前主流派摘要

隐藏：
- 暂停按钮
- 所有次级系统入口

### 8.3 首领战
显示：
- 首领血条
- 波次
- 等级
- 经验
- 气血
- 暂停按钮

隐藏：
- 撤离
- 强化入口常驻按钮

## 9. 微信运行态验收
必须额外验证：
1. 玩家第一次进入战场时，视线是否先落在敌潮推进区与底部防线，而不是按钮区
2. 常规波次时，屏幕上是否不存在常驻底部三按钮
3. 升级弹层是否足够强打断，但关闭后能立即回到战斗节奏
4. 首领战时顶部首领血条是否显著提升威压感
5. 场上敌潮变多时，UI 是否仍能清楚读到波次、等级、经验、气血

## 10. 禁止事项
- 禁止把战场页重新做回三段卡片式后台布局
- 禁止让底部功能按钮主导视觉
- 禁止默认存在储物袋 / 怪物详情 / 撤离三按钮
- 禁止把升级做成边栏小层
- 禁止在战场主画面常驻大段说明文案
