# 轻肉鸽战场UI信息组织对齐方案

## 1. 文档定位
本文档只解决一个问题：
**轻肉鸽战场页在信息组织、交互主次、战斗中弹层与局内反馈上，如何对齐《向僵尸开炮》的实时战斗体验。**

它不定义：
- 波次数值
- 强化内容
- 掉落强度

它定义的是：
- 战场 UI 该怎么摆
- 哪些信息始终可见
- 哪些信息必须让位
- 升级、首领、死亡这些中断节点如何插入

## 2. 第一性原则
如果战场页还是：
- 上中下均分卡片
- 按钮占主视觉
- 信息像表单一样平铺

那即使数值和波次都对，体感也不会像《向僵尸开炮》。

真正对齐的关键不是“长得像”，而是：
1. **战场画面永远是主角**
2. **局内成长是高频插入，而不是主画面常驻大块 UI**
3. **威胁、成长、生命这三类信息始终能快速读懂**

## 3. 正式信息层级
### 3.1 第一层：战场核心层
必须占据绝对主视觉：
- 玩家角色
- 敌潮推进
- 弹道 / 术法
- 首领入场

任何时候都不允许被：
- 大面积说明文字
- 长条任务列表
- 常驻弹框  
压住。

### 3.2 第二层：战斗状态层
始终服务于“我现在打得怎么样”：
- 当前波次
- 当前等级
- 当前经验条
- 当前气血
- 首领血条（出现时）

### 3.3 第三层：操作层
只保留战斗中必要操作：
- 储物袋
- 怪物详情
- 撤离（仅普通波 / 精英波）

### 3.4 第四层：中断层
只在必要时覆盖战场：
- 升级三选一
- 首领登场提示
- 复活确认
- 结算预览

## 4. 正式布局
### 4.1 常规战斗态
推荐结构：
- 左上：波次 + 等级 + 经验
- 上中：关卡名 / 精英提示 / 首领提示
- 右上：暂停 / 设置预留位（如当前版本不做，可空）
- 中央：战场
- 底左：我方气血
- 底右：储物袋 / 怪物详情 / 撤离

### 4.2 首领战
新增：
- 顶部首领血条
- 首领机制短标签

同时隐藏：
- 撤离

### 4.3 升级弹层
升级弹层必须居中覆盖战场，且：
- 一次只出现 3 张卡
- 背后战场暂停
- 弹层关闭后直接回战场，不经过中间页

## 5. 交互主次
### 5.1 战场中玩家的第一决策
不是“点哪个按钮”，而是：
- 走位
- 吃经验
- 看敌潮压力

### 5.2 第二决策
- 升级时选哪张强化卡

### 5.3 第三决策
- 要不要开储物袋
- 要不要查看怪物详情
- 要不要撤离

所以：
**底部功能按钮必须存在，但绝不能成为视觉中心。**

## 6. 按钮与弹层规则
### 6.1 储物袋
- 只能从战场底部次级按钮打开
- 打开后冻结战斗
- 关闭后回到原波次状态

### 6.2 怪物详情
- 只查看当前场上主威胁单位
- 不允许跳成图鉴页

### 6.3 撤离
- `wave / elite` 可见
- `boss_intro / boss / settle_preview` 必须隐藏

### 6.4 升级弹层
- 不允许右侧抽屉
- 不允许底部半屏
- 必须是全局主中断层

## 7. 与《向僵尸开炮》的对齐要求
### 7.1 必须对齐的地方
- 战场始终是主视觉
- 升级是高频插入式中断
- 敌潮压力读感强
- build 成长通过战斗内反馈而不是结算页体现

### 7.2 修仙化改写
不能出现枪械 UI 语义：
- 不出现枪械弹匣感
- 不出现现代军武边框
- 不出现枪械数值词汇

必须转换成修仙语义：
- 波次像妖潮推进
- 强化像悟道 / 升法
- 首领像妖王压场

## 8. 页面显隐硬规则
### 8.1 常规波次
显示：
- 波次
- 等级
- 经验
- 气血
- 储物袋
- 怪物详情
- 撤离

隐藏：
- 旧页式待战按钮
- 神通整备入口
- 结算确认按钮

### 8.2 升级态
显示：
- 3 张强化卡
- 当前等级
- 当前波次
- 当前主流派摘要

隐藏：
- 储物袋
- 怪物详情
- 撤离

### 8.3 首领战
显示：
- 首领血条
- 波次
- 等级
- 经验
- 气血
- 储物袋
- 怪物详情

隐藏：
- 撤离
- 强化入口常驻按钮

## 9. 微信运行态验收
必须额外验证：
1. 玩家第一次进入战场时，视线是否先落在战场中央，而不是按钮区
2. 常规波次时，底部按钮是否是“可用但不抢眼”
3. 升级弹层是否足够强打断，但关闭后能立即回到战斗节奏
4. 首领战时顶部首领血条是否显著提升威压感
5. 场上敌潮变多时，UI 是否仍能清楚读到波次、等级、经验、气血

## 10. 禁止事项
- 禁止把战场页重新做回三段卡片式后台布局
- 禁止让底部功能按钮主导视觉
- 禁止把升级做成边栏小层
- 禁止在战场主画面常驻大段说明文案
- 禁止把储物袋、怪物详情、撤离和升级同时暴露在一个状态里
