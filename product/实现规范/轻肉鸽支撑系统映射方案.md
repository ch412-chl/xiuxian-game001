# 轻肉鸽支撑系统映射方案

## 1. 文档定位
本文档定义轻肉鸽单局结构下，广告、成就、图鉴、纳戒、储物袋、道具、破境如何重新映射。

目标不是重复定义这些系统，而是明确：
- 轻肉鸽版本下它们在什么节点触发
- 写入 runBag 还是 inventory
- 与旧探索主链相比有哪些禁止复活的旧逻辑

## 2. 支撑系统总原则
1. 单局获得的临时收益先进入 runBag
2. 持久收益只在 settle_commit 后写入 inventory / 人物成长
3. 支撑系统不得绕过战场主链直接写永久状态
4. 支撑系统不能继续依赖旧“10层探索日志推进”作为前置

## 3. runBag / inventory / 人物经验

### 3.1 runBag
负责：
- 本局灵石
- 本局道具掉落
- 本局突破材料掉落
- 本局首领奖励预览

### 3.2 inventory
负责：
- 永久纳戒库存
- 已领取成就奖励
- 破境消耗品
- 永久道具/材料/装备

### 3.3 人物经验
负责：
- 本局战斗经验累计结果
- 失败时仍保留经验，不因 runBag 丢失而回滚

## 4. 广告系统映射

### 4.1 广告复活
- 触发节点：本局战斗失败
- 消费位置：轻肉鸽战场页复活确认层
- 成功结果：回到死亡前主战斗态，并恢复部分气血
- 不改变：已获得局内强化、已累计 runBag、已累积经验

### 4.2 广告撤离
- 触发节点：无传送符且当前状态允许主动撤离
- 消费位置：轻肉鸽战场页撤离确认层
- 成功结果：进入 settle_preview_retreat -> settle_commit，保留 runBag 战利品

### 4.3 广告跳冷却
- 不在战场内使用
- 只在人物页破境失败冷却时使用

## 5. 成就系统映射

### 5.1 统计来源
- 击杀普通怪
- 击杀精英
- 击杀首领
- 完成关卡
- 达成境界
- 累计图鉴解锁

### 5.2 写入规则
- 成就进度实时累加
- 成就奖励领取后直接写入 inventory
- 不允许把成就奖励先写入 runBag

## 6. 图鉴系统映射

### 6.1 普通怪图鉴
- 首次遭遇该大境界普通怪时写入

### 6.2 首领图鉴
- 首次进入 boss_intro 时写入
- 不要求首次击杀

### 6.3 精英图鉴
- 当前版本不写一级独立图鉴页
- 仅作为首领条目机制补充和战场认知补充

## 7. 道具系统映射

### 7.1 回灵丹
- 可在战场页通过储物袋使用
- 使用后直接影响本局主战斗态

### 7.2 传送符
- 可在允许撤离状态下使用
- 使用后直接进入撤离成功结算链

### 7.3 突破丹 / 突破材料
- 战场掉落先入 runBag
- 结算完成后写入 inventory
- 只允许在人物页 / 破境流程中消耗

## 8. 破境系统映射
- 破境永远是局外行为
- 轻肉鸽单局只负责提供经验、突破丹、材料
- 不允许在战场结算页直接触发破境确认层

## 9. 事件系统映射
- 轻肉鸽版本不再沿用旧探索日志事件页
- 局内随机补给 / 小强化 / 临时风险，统一视为战场内事件节点
- 这些节点若要进入正式版本，必须额外补战场事件子表

## 12. 当前禁止事项
- 禁止让成就奖励进入 runBag
- 禁止在战场中直接消耗 inventory 之外的临时突破丹完成破境
- 禁止复活旧探索页“储物袋内嵌展开”逻辑
- 禁止把广告撤离写成不经 settle_commit 直接入纳戒
