# 轻肉鸽战斗边界状态机与写回规则

## 1. 文档定位
本文档统一轻肉鸽战斗中的边界状态与写回规则，重点定义：
- 撤离确认
- 广告复活
- 首领过场
- 结算预览
- 正式写回

## 2. 状态定义
### 2.1 主状态
1. `prepare`
2. `wave`
3. `elite`
4. `upgrade`
5. `boss`
6. `settle_preview`

### 2.2 边界状态
1. `retreat_confirm`
2. `revive_offer`
3. `revive_resume`
4. `boss_intro`
5. `settle_commit`
6. `leave_to_role`

## 3. 状态切换唯一口径
### 3.1 开局
`prepare -> wave`

### 3.2 精英波
- 第 5 波：`wave -> elite`
- 第 10 波：`wave -> elite`
- 第 15 波：`wave -> elite`
- 精英不是独立页面，而是战场主状态变化

### 3.3 升级强化
- 玩家在 `wave / elite / boss` 中升级时：`当前主状态 -> upgrade -> 原主状态`
- `upgrade` 与固定精英节点无绑定关系
- 进入 `upgrade` 时必须冻结敌潮、伤害、CD、弹体与计时

### 3.4 首领
`wave(16~19) -> boss_intro -> boss(20)`
- `boss_intro` 只绑定第 20 波开始
- 不允许再绑定“第二次强化确认后”这类旧口径
- `boss_intro` 负责清理残余普通敌潮、展示首领机制、锁定最终阶段提示

### 3.5 撤离
`prepare / wave / elite -> retreat_confirm -> settle_preview_retreat -> settle_commit -> leave_to_role`
- 撤离成功后，不允许直接写回 `inventory`
- 必须先进入 `settle_preview`
- 只有 `settle_commit` 才允许正式写回

### 3.6 死亡与复活
`wave / elite / boss -> revive_offer -> revive_resume -> 原主状态`
`wave / elite / boss -> settle_preview_fail -> settle_commit -> leave_to_role`

唯一口径：
- 满足广告复活条件时先进入 `revive_offer`
- 广告复活成功后回到死亡前主状态
- 认命离开或无复活条件时进入 `settle_preview_fail`
- 失败预览不再展示广告复活或广告撤离

### 3.7 通关
`boss -> settle_preview_win -> settle_commit -> leave_to_role`

## 4. runBag / inventory 唯一写回规则
### 4.1 定义
- `runBag`：本局临时战利品与战场内临时道具变化
- `inventory`：永久纳戒库存

### 4.2 正式写回时机
#### 通关
在 `settle_commit` 中执行：
- `runBag 战利品 -> inventory`
- 本局经验 -> 人物经验
- 本局突破丹 / 灵石 / 道具 -> inventory

#### 主动撤离成功
在 `settle_commit` 中执行：
- `runBag 战利品 -> inventory`
- 本局经验 -> 人物经验

#### 死亡离开
在 `settle_commit` 中执行：
- 本局经验 -> 人物经验
- `runBag` 战利品全部丢失
- `runBag` 清空，不写入 `inventory`

### 4.3 强制边界
1. `settle_preview` 只展示，不做永久写回
2. `settle_commit` 才允许写入 `inventory` 或人物经验
3. 不允许首领死亡当帧直接写回纳戒
4. 不允许撤离确认层广告成功后直接写回纳戒

## 5. 撤离规则
### 5.1 允许撤离的状态
- `prepare`
- `wave`
- `elite`

### 5.2 不允许撤离的状态
- `upgrade`
- `boss_intro`
- `boss`
- `settle_preview`
- `settle_commit`

## 6. 广告复活规则
### 6.1 进入条件
- 玩家死亡
- 本局尚未使用广告复活
- 当前广告配置允许复活
- 当前主状态属于 `wave / elite / boss`

### 6.2 revive_resume 规则
- 回复固定比例气血
- 返回死亡前主状态
- runBag 保留
- 局内强化保留
- 不允许回到旧待战态或旧探索页

### 6.3 广告失败兜底
- 广告失败则停留在 `revive_offer`
- 不自动判定撤离
- 不自动写回战利品
- 不自动进入失败结算

## 7. Boss 过场规则
### 7.1 boss_intro 必做事项
1. 清理普通敌潮
2. 冻结无关输入
3. 展示首领名称与两条机制提示
4. 提示第 20 波终局开始

### 7.2 boss_intro 禁止事项
1. 不允许继续刷普通波次
2. 不允许弹出储物袋或怪物详情
3. 不允许显示撤离按钮
4. 不允许跳过过场直接进入 boss

## 8. 同帧优先级
若同一帧发生首领死亡与玩家死亡：
1. 先判断首领是否已正式死亡
2. 若首领已死亡，则判定为通关优先
3. 进入 `settle_preview_win`
4. 不再触发广告复活

- 死亡离开经验是否保留
