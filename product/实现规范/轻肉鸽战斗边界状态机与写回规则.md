# 轻肉鸽战斗边界状态机与写回规则

## 1. 文档定位
本文档补齐轻肉鸽战斗中“主流程之外”的边界状态，统一：
- 撤离确认
- 广告复活
- Boss 过场
- 结算预览
- 正式写回

目标是让实现线程不再靠猜处理 `runBag`、复活与结算。

## 2. 主状态与边界状态

### 2.1 主状态
1. `prepare`
2. `wave`
3. `elite`
4. `upgrade`
5. `boss`
6. `settle_preview`

### 2.2 边界状态
1. `retreat_confirm`
2. `revive_offer`
3. `revive_resume`
4. `boss_intro`
5. `settle_commit`
6. `leave_to_role`

## 3. 状态切换唯一口径

### 3.1 开局
`prepare -> wave`
- 玩家点击开始或倒计时结束后进入首轮敌潮

### 3.2 精英
`wave -> elite`
- 到达首个精英时间点时进入 `elite`
- 精英不是独立页面，而是战场主状态变化

### 3.3 强化
`elite -> upgrade`
- 对应精英节点清除后进入 `upgrade`
- **进入 `upgrade` 时战斗必须完全暂停**
- 敌潮、计时、CD、弹体全部冻结

### 3.4 Boss
`upgrade -> boss_intro -> boss`
- 第二次强化后先进入 `boss_intro`
- `boss_intro` 负责：
  - 清理残余普通怪
  - 播放首领登场提示
  - 锁定战场镜头/标题提示
- 过场结束才进入 `boss`

### 3.5 结算
`boss -> settle_preview -> settle_commit -> leave_to_role`
- 首领死亡后先进入 `settle_preview`
- 只展示本局结果，不直接永久写回
- 玩家点击确认后进入 `settle_commit`
- `settle_commit` 执行正式写回

## 4. runBag / inventory 唯一写回规则

### 4.1 当前定义
- `runBag`：本局临时战利品与本局临时道具变化
- `inventory`：永久纳戒库存

### 4.2 正式写回时机

#### 通关
- 通关后先进入 `settle_preview`
- 玩家确认结算时进入 `settle_commit`
- 在 `settle_commit` 中执行：
  - `runBag 战利品 -> inventory`
  - 本局经验 -> 人物经验
  - 本局掉落的突破丹 / 灵石 / 道具 -> inventory`

#### 主动撤离成功
- `wave / elite / prepare` 等允许撤离的状态进入 `retreat_confirm`
- 确认撤离成功后直接进入 `settle_preview`
- 后续与通关一致，在 `settle_commit` 中正式写回

#### 死亡离开
- 死亡且不复活时，不进入通关结算
- 进入失败结算预览后：
  - 保留本局已获得经验
  - 丢失 `runBag` 中标记为战利品的掉落
  - 不写回战利品到 `inventory`

### 4.3 当前强制边界
- `settle_preview` 只展示，不做永久写回
- `settle_commit` 才允许写入 `inventory`
- 不允许在 Boss 死亡当帧直接写回纳戒

## 5. 撤离规则

### 5.1 允许撤离的状态
- `prepare`
- `wave`
- `elite`

### 5.2 不允许撤离的状态
- `upgrade`
- `boss_intro`
- `boss`
- `settle_preview`
- `settle_commit`

### 5.3 撤离确认层必须显示
- 当前关卡名
- 当前生存状态
- 将保留什么
- 将丢失什么
- 是否消耗传送符 / 是否允许广告撤离

## 6. 广告复活规则

### 6.1 进入条件
- 玩家死亡
- 本局尚未使用广告复活
- 当前广告配置允许复活

### 6.2 状态流
`wave / elite / boss -> revive_offer -> revive_resume -> 原主状态`

### 6.3 revive_resume 规则
- 回复固定比例气血
- 恢复到死亡前的主状态：
  - 在 `wave` 死亡，回 `wave`
  - 在 `elite` 死亡，回 `elite`
  - 在 `boss` 死亡，回 `boss`
- 不允许回到旧“遇敌待战”或旧探索页

### 6.4 广告复活失败兜底
- 广告失败则停留在失败确认层
- 不自动判定撤离
- 不自动写回战利品

## 7. Boss 过场规则

### 7.1 进入 Boss 前
- 第二次强化确认后，不立即开打
- 必须进入 `boss_intro`

### 7.2 boss_intro 必须做的事
- 清理普通敌潮
- 冻结玩家输入外的无关系统
- 展示首领名称与机制提示
- 提示本局将进入最终阶段

### 7.3 boss_intro 禁止事项
- 不允许继续刷普通波次
- 不允许弹出其他页面
- 不允许撤离按钮继续可见

## 8. 同帧优先级
若同一帧发生：
- 首领死亡
- 玩家死亡

唯一口径：
1. 先判断首领是否已正式死亡
2. 若首领已死亡，则判定为通关优先
3. 进入 `settle_preview`
4. 不再触发广告复活

## 9. 与旧探索战斗的替换关系
- 轻肉鸽主链不再使用“遇敌待战”作为战斗入口
- 旧探索战斗只作为当前旧实现参考
- 新战斗实现必须以本状态机为准

## 10. 微信运行态验收强依赖
后续实现线程必须用：
- `轻肉鸽微信运行态验收补充`
来验证以下边界：
- 升级暂停是否冻结
- 广告复活是否回原状态
- 撤离是否合法
- 结算写回是否只在 `settle_commit`

## 11. 实现前若仍缺信息
若仍无法唯一回答以下问题：
- 死亡离开经验是否保留
- 撤离写回时点
- Boss 前残怪是否清理
- 广告失败后停在哪里

则只能标缺口，不能猜写。
