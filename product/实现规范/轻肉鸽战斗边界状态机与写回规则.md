# 轻肉鸽战斗边界状态机与写回规则

## 1. 文档定位
本文档补齐轻肉鸽战斗中“主流程之外”的边界状态，统一：
- 撤离确认
- 广告复活
- Boss 过场
- 结算预览
- 正式写回

目标是让实现线程不再靠猜处理 `runBag`、复活、撤离与结算。

## 2. 状态定义

### 2.1 主状态
1. `prepare`
2. `wave`
3. `elite`
4. `upgrade`
5. `boss`
6. `settle_preview`

### 2.2 边界状态
1. `retreat_confirm`
2. `revive_offer`
3. `revive_resume`
4. `boss_intro`
5. `settle_commit`
6. `leave_to_role`

### 2.3 展示变体说明
以下名字只用于页面展示分支，不额外新增独立主状态：
1. `settle_preview_win`
2. `settle_preview_retreat`
3. `settle_preview_fail`

补充口径：
- `revive_offer` 对应页面层：复活确认层
- `retreat_confirm` 对应页面层：撤离确认层
- `leave_to_role` 是离开单局的技术终态，不等于“回人物页”；当前默认落点为历练选关页

## 3. 状态切换唯一口径

### 3.1 开局
`prepare -> wave`
- 玩家点击开始或倒计时结束后进入首轮敌潮

### 3.2 精英
`wave -> elite`
- 到达精英节点时进入 `elite`
- 精英不是独立页面，而是战场主状态变化

### 3.3 强化
`elite -> upgrade`
- 对应精英节点清除后进入 `upgrade`
- 进入 `upgrade` 时战斗必须完全暂停
- 敌潮、计时、CD、弹体全部冻结

### 3.4 Boss
`upgrade -> boss_intro -> boss`
- 第二次强化确认后先进入 `boss_intro`
- `boss_intro` 负责：
  - 清理残余普通怪
  - 播放首领登场提示
  - 锁定战场镜头 / 标题提示
- 过场确认继续后才进入 `boss`

### 3.5 撤离
`prepare / wave / elite -> retreat_confirm -> settle_preview_retreat -> settle_commit -> leave_to_role`
- 撤离确认成功后，不直接写回 `inventory`
- 仍必须先进入 `settle_preview`
- 只有 `settle_commit` 才允许正式写回

### 3.6 失败与复活
`wave / elite / boss -> revive_offer -> revive_resume -> 原主状态`
`wave / elite / boss -> settle_preview_fail -> settle_commit -> leave_to_role`

唯一口径：
- 玩家死亡后，若满足广告复活条件，则先进入 `revive_offer`
- 玩家在 `revive_offer` 选择广告复活成功，进入 `revive_resume` 后回到死亡前主状态
- 玩家在 `revive_offer` 选择认命离开，或本局根本不满足复活条件，则进入 `settle_preview_fail`
- `settle_preview_fail` 不再展示广告复活或广告撤离

### 3.7 通关
`boss -> settle_preview_win -> settle_commit -> leave_to_role`
- 首领死亡后先进入 `settle_preview_win`
- 只展示本局结果，不直接永久写回
- 玩家确认后进入 `settle_commit`

## 4. runBag / inventory 唯一写回规则

### 4.1 当前定义
- `runBag`：本局临时战利品与本局临时道具变化
- `inventory`：永久纳戒库存

### 4.2 正式写回时机

#### 通关
- 在 `settle_commit` 中执行：
  - `runBag 战利品 -> inventory`
  - 本局经验 -> 人物经验
  - 本局掉落的突破丹 / 灵石 / 道具 -> inventory

#### 主动撤离成功
- 在 `settle_commit` 中执行：
  - `runBag 战利品 -> inventory`
  - 本局经验 -> 人物经验
- 撤离确认层与结算预览页都不允许直接写回

#### 死亡离开
- 在 `settle_commit` 中执行：
  - 本局经验 -> 人物经验
  - `runBag` 中标记为战利品的掉落全部丢失
  - `runBag` 清空，不写入 `inventory`

### 4.3 当前强制边界
- `settle_preview` 只展示，不做永久写回
- `settle_commit` 才允许写入 `inventory` 或人物经验
- 不允许在 Boss 死亡当帧直接写回纳戒
- 不允许在撤离确认层广告成功后直接写回纳戒

## 5. 撤离规则

### 5.1 允许撤离的状态
- `prepare`
- `wave`
- `elite`

### 5.2 不允许撤离的状态
- `upgrade`
- `boss_intro`
- `boss`
- `settle_preview`
- `settle_commit`

### 5.3 撤离确认层必须显示
- 当前关卡名
- 当前主状态
- 将保留什么
- 将丢失什么
- 是否消耗传送符
- 是否允许广告撤离

### 5.4 按钮显隐唯一口径
- 有传送符：显示 `使用传送符撤离`
- 无传送符且广告可用：显示 `广告撤离`
- 两者都不可用：只允许 `继续历练`
- 不允许在 `boss` 中通过任何方式打开撤离确认层

## 6. 广告复活规则

### 6.1 进入条件
- 玩家死亡
- 本局尚未使用广告复活
- 当前广告配置允许复活
- 当前主状态属于 `wave / elite / boss`

### 6.2 状态流
`wave / elite / boss -> revive_offer -> revive_resume -> 原主状态`

### 6.3 revive_resume 规则
- 回复固定比例气血
- 恢复到死亡前的主状态：
  - 在 `wave` 死亡，回 `wave`
  - 在 `elite` 死亡，回 `elite`
  - 在 `boss` 死亡，回 `boss`
- runBag 保留
- 局内强化保留
- 不允许回到旧“遇敌待战”或旧探索页

### 6.4 广告复活失败兜底
- 广告失败则停留在 `revive_offer`
- 不自动判定撤离
- 不自动写回战利品
- 不自动进入 `settle_preview_fail`

## 7. Boss 过场规则

### 7.1 进入 Boss 前
- 第二次强化确认后，不立即开打
- 必须进入 `boss_intro`

### 7.2 boss_intro 必须做的事
- 清理普通敌潮
- 冻结玩家输入外的无关系统
- 展示首领名称与两条机制提示
- 提示本局将进入最终阶段

### 7.3 boss_intro 禁止事项
- 不允许继续刷普通波次
- 不允许弹出其他页面
- 不允许撤离按钮继续可见
- 不允许跳过本层直接进入 `boss`

## 8. 同帧优先级
若同一帧发生：
- 首领死亡
- 玩家死亡

唯一口径：
1. 先判断首领是否已正式死亡
2. 若首领已死亡，则判定为通关优先
3. 进入 `settle_preview_win`
4. 不再触发广告复活

## 9. 与页面层的对应关系
- `retreat_confirm` 只对应撤离确认层，不对应失败页
- `revive_offer` 只对应复活确认层，不对应结算预览页
- `settle_preview_fail` 只展示失败结果，不再承担广告补救入口
- `settle_commit` 完成后统一离开战斗主链，默认回流历练选关页

## 10. 微信运行态验收强依赖
后续实现线程必须用：
- `轻肉鸽微信运行态验收补充.md`
- `轻肉鸽页面职责与交互验收矩阵.md`

来验证以下边界：
- 升级暂停是否冻结
- 广告复活是否回原状态
- 撤离是否合法
- 失败预览是否不再承接广告补救
- 结算写回是否只在 `settle_commit`
