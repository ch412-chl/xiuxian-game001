实现说明

1. 结构主次

  • 中部关卡列表必须是绝对主视觉。
  • 玩家第一眼应该看到“主推哪一关”，而不是技能、资源或说明文字。
  • 每次只展示当前境界的 3 个关卡，不做长列表。

2. 每张关卡卡片必须包含

  • 关卡名称
  • 当前状态：主推 / 已通 / 未达
  • 一句敌潮风格摘要
  • 一句地图摘要
  • 唯一主按钮：`进入战斗` 或不可用说明

3. 交互逻辑

点击已解锁关卡卡片 -> 直接进入轻肉鸽战场页 点击技能页入口 -> 进入技能页 点击未解锁关卡卡片 -> 只提示解锁条件，不跳转 底部主导航固定可用 本页不允许展开神通详情和 runBag

4. 图片与视觉占位

  • 当前无需真实图片，但可在卡片左侧预留关卡意象缩略图位。
  • 实现线程若看到后续插图占位，应理解为“辅助氛围”，不是点击热区。

5. 数据依赖

读取： - stageId - 大境界 / 阶段 / 关卡名称 - themeTag / themeDesc - battleMapId - modeParamId 来源： 轻肉鸽关卡映射总表.csv
