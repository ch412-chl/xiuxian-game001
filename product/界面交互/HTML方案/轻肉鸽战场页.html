实现说明

1. 第一眼必须看到什么

  • 敌潮推进区与底部防线，而不是按钮。
  • 顶部 HUD 是辅助读数，不是主视觉中心。
  • 战场中唯一高频交互是升级三选一卡牌。

2. UI 层级

第一层：战场核心层（敌潮 / 玩家防线 / 自动术法） 第二层：状态层（波次 / 等级 / 经验 / 气血 / 首领血条） 第三层：暂停入口（极弱存在） 第四层：中断层（升级 / 首领过场 / 复活 / 结算 / 暂停菜单）

3. 关键显隐

  • `wave`：仅显示 HUD 与暂停按钮，第 5 / 10 / 15 波只改变怪物生成组合。
  • `boss`：增加首领血条，仍不显示常驻撤离按钮。
  • `upgrade`：隐藏暂停按钮，只留升级弹层。

4. 图片占位如何理解

  • 战场背景图是大氛围层，不是可点区域。
  • 玩家底部法阵 / 立绘属于防线表现层，不等于页面按钮。
  • 首领立绘 / 敌潮剪影属于战场表现层，不应当替代逻辑区域。

5. 对齐要求

必须对齐《向僵尸开炮》的地方： - 玩家固定在底部 - 自动攻击 - 敌潮持续压近 - 升级三选一卡牌高频中断 - 波次只影响生成，不清理场内残怪 必须修仙化的地方： - 技能语汇 - 敌人题材 - 场景意象 - 卡牌文案
