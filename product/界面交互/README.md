# 界面交互

## 目录职责
本目录只存放当前版本的 UI/UE 总控文档、页面交互导航、视觉方向稿与线框。

当前职责：
- `界面交互规范.md`：当前版本 UI/UE 总控文档，定义全局布局、组件规则、页面层级与交互红线
- `页面交互导航.md`：页面级入口，告诉实现线程该读哪份实现规范与哪张线框
- `视觉方向方案.md`：当前版本视觉气质真源，定义整体配色、材质、字体语气与页面视觉方向
- `视觉方向提案.html`：当前版本视觉方向浏览器预览稿，用于快速拍板视觉方向
- `页面视觉深化稿.html`：当前版本页面级视觉深化稿，用于查看历练页与探索战斗页更接近最终状态的页面气质
- `界面交互线框.html`：当前版本线框与页面分区示意

## 读取时机
出现以下任务时优先读取本目录：
- 调整页面布局、交互流转或显隐规则
- 判断某页面该展示什么、不该展示什么
- 核对实现是否偏离当前 UI/UE 口径

## 阅读顺序
1. 先读 `界面交互规范.md`
2. 再读 `页面交互导航.md`
3. 若任务涉及整体视觉与调色，读 `视觉方向方案.md`
4. 若要判断页面气质是否到位，优先看 `页面视觉深化稿.html`
5. 再看 `视觉方向提案.html` 与 `界面交互线框.html`
6. 若需要代码落地约束，回看 `../实现规范/`

## 真源说明
本目录属于交互规则真源层。
- 本目录不负责定义玩法和数值真源
- 若交互规则与总 PRD 冲突，以 `../产品需求/当前产品需求.md` 为产品方向真源
- 若交互规则与代码现状冲突，应优先检查实现规范与当前实现是否失真

## 使用约束
- 不把本目录当作数值或玩法定义目录使用
- 不跳过本目录直接凭当前 UI 现状推断正确交互
- 若要调整主链交互，先核对本目录与 `../实现规范/`

## 维护规则
满足以下情况时，增量更新本 README：
- 新增了本目录正式交互文件
- 调整了本目录文件职责或阅读顺序
- 确认了新的长期有效交互边界

以下情况不要更新本 README：
- 只是代码层的小修复
- 只是临时讨论方案
- 只是 `HANDOFF.md` 中的阶段状态变化
