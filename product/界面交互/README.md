# 界面交互

## 目录职责

本目录只存放当前版本的 UI/UE 规格与线框。

当前职责：
- `界面交互规范.md`：当前版本正式交互规则
- `界面交互线框.html`：当前版本线框与页面分区示意

## 读取时机

出现以下任务时优先读取本目录：
- 调整页面布局、交互流转或显隐规则
- 判断某页面该展示什么、不该展示什么
- 核对实现是否偏离当前 UI/UE 口径

## 阅读顺序

1. 先读 `界面交互规范.md`
2. 再看 `界面交互线框.html`
3. 若需要代码落地约束，回看 `../实现规范/`

## 真源说明

本目录属于交互规则真源层。
- 本目录不负责定义玩法和数值真源
- 若交互规则与总 PRD 冲突，以 `../产品需求/当前产品需求.md` 为产品方向真源
- 若交互规则与代码现状冲突，应优先检查实现规范与当前实现是否失真

## 当前有效文件

- `界面交互规范.md`：正式交互规格
- `界面交互线框.html`：页面线框与分区示意

## 使用约束

- 不把本目录当作数值或玩法定义目录使用
- 不跳过本目录直接凭当前 UI 现状推断正确交互
- 若要调整主链交互，先核对本目录与 `../实现规范/`

## 维护规则

满足以下情况时，增量更新本 README：
- 新增了本目录正式交互文件
- 调整了本目录文件职责或阅读顺序
- 确认了新的长期有效交互边界

以下情况不要更新本 README：
- 只是代码层的小修复
- 只是临时讨论方案
- 只是 `HANDOFF.md` 中的阶段状态变化
