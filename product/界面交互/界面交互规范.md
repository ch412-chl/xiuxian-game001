## 3. 全局设计约束
### 3.1 画布与布局
- 目标设备比例：手机竖屏
- 主内容区采用单列布局
- 页面采用上中下三段结构
- 同屏优先只保留当前决策所需信息

### 3.2 页面分区比例
以页面可用高度 `H` 为基准：

| 页面 | 顶部区 | 中部区 | 底部区 |
| --- | --- | --- | --- |
| 人物 | 24% | 52% | 24% |
| 历练选关 | 18% | 70% | 12% |
| 战前整备 | 18% | 70% | 12% |
| 探索未遇敌 | 14% | 48% | 38% |
| 探索遇敌 | 20% | 48% | 32% |
| 洞府 | 18% | 70% | 12% |
| 纳戒 | 12% | 88% | 0% |
| 成就 | 12% | 88% | 0% |
| 妖物图鉴 | 12% | 88% | 0% |
| 储物袋 | 12% | 88% | 0% |

### 3.3 基础尺寸
- 页面左右安全边距：16 px
- 最小点击高度：44 px
- 顶部标题区建议高度：48 px
- 底部按钮区建议高度：52 px
- 神通 / 技能卡格最小高度：52 px
- 弹框最小宽度：220 px
- 血条最小高度：10 px
- 列表卡片最小高度：40 px
- 顶部标题与副标题垂直间距：8 px

### 3.4 空间压缩优先级
当空间不足时，压缩顺序统一如下：
1. 说明性留白
2. 中部提示区
3. 卡格间距
4. 启用内部滚动区

禁止压缩：
- 顶部安全区
- 血条高度
- 按钮点击高度
- 左右安全边距

### 3.5 通用显示规则
- 0 数量内容不展示
- 未遭遇敌人不展示敌方区
- 未发生技能释放不展示 CD 区
- 外层不展示攻击 / 防御 / 神识
- 储物袋为独立页面，不作为探索页内弹框展示
- 成就与图鉴入口只从洞府进入

### 3.6 热区与信息优先级
- 关卡行整行可点
- 神通格 / 技能格整格可点
- 返回箭头点击区不得小于 44 px 高
- 血条热区只承担详情入口，不得与底部按钮重叠
- 当空间不足时，信息让位顺序为：
  1. 说明性文案
  2. 中部留白
  3. 卡格间距
  4. 中部内部滚动

## 5. 页面级视觉重点
### 5.1 人物页
- 主重点：当前境界、经验进度、破境状态
- 次重点：纳戒入口、当前目标摘要
- 禁止做成纯属性陈列页

### 5.2 历练选关页
- 主重点：当前该打哪一关
- 次重点：当前 3 槽整备摘要
- 禁止让神通区挤占关卡列表主视觉

### 5.3 战前整备页
- 主重点：3 个出手顺序槽
- 次重点：当前可替换神通
- 禁止在整备页承担选关主职责

### 5.4 探索页
- 未遇敌：顶部关卡信息、中部短时提示、底部人物信息与操作按钮
- 遇敌待战：顶部敌方信息、中部战斗舞台、底部我方信息 + 待战按钮 + 神通格
- 战斗中：隐藏待战按钮，只保留血条与神通格
- 禁止恢复战斗日志、长播报文字流

### 5.5 洞府页
- 主重点：阶段驾驶舱与功能入口
- 次重点：成就 / 图鉴 / 阶段目标摘要
- 禁止恢复洞府生产链旧结构

### 5.6 纳戒 / 储物袋
- 纳戒：永久库存页
- 储物袋：单次历练库存页
- 两页都应列表化，不做弹层化伪页面

### 5.7 成就 / 图鉴
- 成就：进度与领奖反馈页
- 图鉴：遭遇解锁与档案查看页
- 禁止使用伪滚动交互

## 6. 通用组件规范
### 6.1 按钮
- 文本居中
- 禁用态置灰
- 点击态必须有即时高亮反馈

### 6.2 卡格
- 统一描边
- 统一内边距
- 统一点击区域
- 神通格 / 技能格优先服务操作，不服务收藏展示

### 6.3 弹框
- 点击空白关闭
- 内容居中
- 不展示冗余说明
- 弹框只承载主界面未展示的信息

### 6.4 血条
- 血条数字置于血条内部
- 敌我血条样式一致
- 战斗受击时允许抖动与拖尾，但不能影响读数

## 7. 交互红线
- 不允许在未确认方案前直接修改主链交互
- 不允许恢复伪滚动方案
- 不允许将储物袋展开到探索页主体中
- 不允许在战斗中保留“撤离 / 战斗 / 储物袋”按钮
- 不允许重新外显旧属性和旧战斗日志

## 9. 交互反馈规范
- 按钮点击：0~100ms 内出现高亮反馈
- 弹框打开：200ms 内完成
- 短时文本：出现后停留再自动消失
- 受击反馈：与血条变化同步
- 冷却反馈：CD 条连续更新，不得跳帧式闪烁

- 历练选关页
- 战前整备页
- 探索页未遇敌态
- 探索页遇敌待战态
- 探索页战斗态
- 怪物详情弹框
- 储物袋页
- 洞府页
- 纳戒页
- 成就页
- 妖物图鉴页

## 11. 前端落地要求
前端实现需按以下优先级落地：
1. 页面框架与区块位置
2. 状态切换与显隐逻辑
3. 点击热区与跳转关系
4. 卡格、按钮、弹框统一组件化
5. 动效与反馈补充

## 12. 空间利用验收口径
- 页面主信息区占比必须高于说明性留白
- 同屏不得出现 3 个以上低优先级信息区同时可见
- 探索页顶部、中部、底部三分区必须稳定
- 历练选关页和战前整备页必须职责分明
- 洞府页标题区、功能入口区、主导航区必须层级清楚
