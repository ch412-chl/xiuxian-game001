# T01 真源桥接补完建议

## 1. 结论先行
当前轻肉鸽真源的真正断裂点不是“表不够多”，而是**主线关卡、轻肉鸽关卡、敌群、精英、首领、图鉴、掉落之间没有稳定桥字段**。在现有材料下，必须先补 4 张中间主表，并对 4 张旧表增补桥字段，否则实现线程只能继续靠名称硬匹配或运行时临时映射。

---

## 2. 需要新增的中间主表清单

### 2.1 轻肉鸽关卡映射总表
作用：把主线关卡真源与轻肉鸽 stageId 建立唯一映射，成为波次表、首领表、图鉴、掉落的总入口。

至少字段：
- `stageId`
- `所属大境界`
- `所属阶段`
- `主线关卡名称`
- `主线关卡顺序`
- `轻肉鸽关卡标题`
- `主题标签`
- `普通怪enemyId`
- `小Boss enemyId`
- `大Boss enemyId`
- `图鉴普通怪ID`
- `图鉴BossID`
- `首领机制bossId`
- `默认掉落组ID`
- `默认波次配置Key`
- `是否当前已配置完成`

### 2.2 轻肉鸽敌群组成总表
作用：正式解释 `enemyGroupId`，把“这一波到底刷什么、怎么混编、怎么形成压迫”真源化。

至少字段：
- `enemyGroupId`
- `stageId`
- `敌群名称`
- `主题标签`
- `普通怪enemyId`
- `特种怪enemyId`
- `普通怪权重`
- `特种怪权重`
- `单批生成数`
- `危险单位上限`
- `刷怪入口规则`
- `压迫意图`
- `是否允许与精英共存`
- `备注`

说明：当前资料里没有“特种怪”正式主表，因此 `特种怪enemyId` 允许先为空；若为空，必须在实现前补表，不能运行时发明。

### 2.3 轻肉鸽精英配置总表
作用：补齐 `eliteId` 的唯一真源，并让精英能桥接图鉴、掉落、波次奖励节点。

至少字段：
- `eliteId`
- `stageId`
- `精英名称`
- `精英档案标题`
- `继承普通怪enemyId`
- `独立图鉴ID`
- `入场波次ID`
- `入场规则`
- `核心机制名称`
- `核心机制描述`
- `精英掉落组ID`
- `击败奖励节点`
- `是否写入图鉴`
- `是否计入首领战前置`
- `备注`

说明：当前资料不足以确认“精英是否必须进入图鉴”是正式规则还是建议规则，因此该字段必须保留，不能省略。

### 2.4 轻肉鸽掉落组配置总表
作用：补齐 `dropGroupId` / `rewardDropGroup`，统一首领、精英、结算写回来源。

至少字段：
- `dropGroupId`
- `所属stageId`
- `掉落来源类型`
- `来源ID`
- `奖励1类型`
- `奖励1数量`
- `奖励2类型`
- `奖励2数量`
- `奖励3类型`
- `奖励3数量`
- `写入位置`
- `失败是否丢失`
- `是否受广告加成`
- `备注`

说明：当前资料里还没有正式掉落内容表，因此奖励类型与数量只能先留结构，不应在实现时硬填。

## 3. 各表之间的桥字段关系

```text
主线关卡经验表
  -> 轻肉鸽关卡映射总表(stageId, 所属大境界, 所属阶段, 主线关卡名称)
  -> 轻肉鸽敌潮波次配置总表(stageId)
  -> 轻肉鸽首领机制配置总表(stageId, bossId)

敌人配置总表(enemyId 新增后)
  -> 轻肉鸽关卡映射总表(普通怪enemyId / 小Boss enemyId / 大Boss enemyId)
  -> 轻肉鸽敌群组成总表(普通怪enemyId / 特种怪enemyId)
  -> 轻肉鸽精英配置总表(继承普通怪enemyId)

妖物图鉴总表(图鉴ID)
  -> 轻肉鸽关卡映射总表(图鉴普通怪ID / 图鉴BossID)
  -> 轻肉鸽精英配置总表(独立图鉴ID)
  -> 轻肉鸽首领机制配置总表(建议补 atlasId)

轻肉鸽敌潮波次配置总表
  -> 轻肉鸽敌群组成总表(enemyGroupId)
  -> 轻肉鸽精英配置总表(eliteId)
  -> 轻肉鸽掉落组配置总表(rewardNode 如需结算奖励时再桥接)

轻肉鸽首领机制配置总表
  -> 轻肉鸽关卡映射总表(stageId, bossId)
  -> 轻肉鸽掉落组配置总表(rewardDropGroup)
  -> 妖物图鉴总表(atlasId 建议补)
```

### 3.1 当前必须形成唯一映射的桥字段
- `stageId`
- `enemyId`
- `enemyGroupId`
- `eliteId`
- `bossId`
- `图鉴ID`
- `dropGroupId`

### 3.2 当前明确禁止的桥接方式
- 通过中文名称硬匹配
- 通过“同大境界 + 同阶段”默认推断
- 通过 `bossName` 或 `显示名称` 猜测图鉴归属
- 通过 `waveType = boss` 默认推出掉落组

## 4. 哪些旧表字段要增补

### 4.1 敌人配置总表.csv 必须增补
当前缺少唯一敌人键，无法被新中间主表稳定引用。

建议至少增补：
- `enemyId`
- `stageId`
- `atlasId`
- `默认dropGroupId`
- `是否可作为精英继承源`

### 4.2 妖物图鉴总表.csv 必须增补
当前只能覆盖普通怪 / Boss，且无法回链精英与首领机制。

建议至少增补：
- `enemyId`
- `bossId`
- `eliteId`
- `stageId`
- `是否精英图鉴`
- `解锁写回规则`

说明：`enemyId / bossId / eliteId` 不要求同一行同时填满，但字段必须存在。

### 4.3 轻肉鸽敌潮波次配置总表.csv 必须增补
当前只有时间轴和代号，缺能支撑体验与桥接的字段。

建议至少增补：
- `themeTag`
- `pressureType`
- `spawnLaneRule`
- `enemyGroupRef`（如与 `enemyGroupId` 同义，可保留一项，但必须统一命名）
- `specialEnemyCap`
- `bossId`
- `wxAcceptanceCase`

说明：`boss` 波目前只靠 `waveType = boss` 区分，不足以建立唯一首领桥接。

### 4.4 轻肉鸽首领机制配置总表.csv 必须增补
当前只有名字、两条机制文案、阈值、掉落组，仍然不足以支撑实现。

建议至少增补：
- `atlasId`
- `enemyId`（或明确为 `bossEnemyId`）
- `phase1Trigger`
- `phase2Trigger`
- `summonEnemyGroupId`
- `enrageBehavior`
- `mechanicTelegraph`
- `wxAcceptanceCase`

## 5. 实现前缺失项

以下内容在当前材料中仍然缺失，**实现线程不能猜着补**：

### 5.1 缺失的正式掉落内容
- `drop_lq_early_boss`、`drop_zj_early_boss` 等掉落组只有 ID，没有正式内容。
- 当前只能建议新增掉落组主表，不能推断具体掉什么、掉多少。

### 5.2 缺失的精英真源
- `elite_lq_early_01`、`elite_lq_early_02` 只有引用，没有正式主表。
- 无法确认名称、机制、图鉴归属、掉落归属、是否独立档案。

### 5.3 缺失的敌群组成规则
- `LQ_EARLY_01_W1_PACK` 等只有 groupId，没有组成与刷怪口定义。
- 无法确认同一敌群是否混入远程、突进、护盾、召唤单位。

### 5.4 缺失的 stageId 命名映射规范
- 当前只看到 `LQ_EARLY_01 / ZJ_EARLY_01` 这类代号。
- 尚无正式文档定义：大境界缩写、阶段缩写、序号规则是否固定。

### 5.5 缺失的图鉴覆盖边界
- 目前图鉴表只覆盖普通怪与 Boss。
- 没有正式说明精英是否进入图鉴、召唤物是否进入图鉴、小Boss是否单列图鉴。

### 5.6 缺失的 rewardNode 语义表
- `upgrade_1 / upgrade_2 / clear` 已出现，但没有正式语义表。
- 无法确认它们分别代表：强化选择点、结算点、掉落发放点，还是三者兼有。

## 6. 最优落地顺序
1. 先给 `敌人配置总表.csv` 补 `enemyId + stageId`。
2. 新增 `轻肉鸽关卡映射总表.csv`，把主线关卡与轻肉鸽 stageId 固定下来。
3. 新增 `轻肉鸽敌群组成总表.csv` 与 `轻肉鸽精英配置总表.csv`。
4. 新增 `轻肉鸽掉落组配置总表.csv`。
5. 再回补 `妖物图鉴总表.csv / 轻肉鸽敌潮波次配置总表.csv / 轻肉鸽首领机制配置总表.csv` 的桥字段。

---

## 7. 一句话结论
T01 的核心不是“再加几张表”，而是**先建立稳定的 `stageId -> enemyId / enemyGroupId / eliteId / bossId / 图鉴ID / dropGroupId` 桥接链**。在这条链未补齐前，后续任何轻肉鸽实现都不应进入代码。
