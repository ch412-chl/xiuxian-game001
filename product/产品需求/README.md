# 产品需求

## 目录职责

本目录只存放当前版本的总需求文档与修订日志。

当前职责：
- `当前产品需求.md`：当前版本唯一正式 PRD
- `修订日志.md`：追踪当前版本需求变更历史

## 读取时机

出现以下任务时优先读取本目录：
- 需要确认当前版本产品范围
- 需要判断玩法边界或模块状态
- 需要核对某项实现是否偏离正式 PRD

## 阅读顺序

1. 先读 `当前产品需求.md`
2. 再看 `修订日志.md`
3. 若需要进入代码实现层，再去 `../实现规范/`

## 真源说明

本目录属于产品方向真源层。
- 本目录不再放实现级方案
- 实现线程不要把本目录当作“代码落地细节说明”目录使用
- 若本目录与局部交互或实现规范冲突，以本目录判断产品方向

## 当前有效文件

- `当前产品需求.md`：当前版本正式 PRD
- `修订日志.md`：当前版本需求变更记录

## 使用约束

- 不跳过本目录直接按个人理解扩玩法
- 不把实现级细节回写到本目录
- 若正式产品口径发生变化，优先更新本目录，再同步其他目录说明

## 维护规则

满足以下情况时，增量更新本 README：
- 新增或替换了本目录正式真源文件
- 调整了本目录文件职责或读取顺序
- 明确了新的长期有效边界

以下情况不要更新本 README：
- 只是局部代码修复
- 只是阶段性接力变化
- 只是会话过程记录
