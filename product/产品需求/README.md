# 产品需求

## 目录职责
本目录只存放当前版本的产品总控文档、模块导航与修订日志。

当前职责：
- `当前产品需求.md`：当前版本总控 PRD，只定义全局目标、边界、真源关系与验收原则
- `模块需求导航.md`：模块级阅读入口，告诉实现线程应该进入哪份实现规范和哪张主表
- `修订日志.md`：追踪当前版本需求变更历史

## 读取时机
出现以下任务时优先读取本目录：
- 需要确认当前版本范围与边界
- 需要判断某个模块该看哪份实现规范和哪张主表
- 需要判断某项实现是否偏离正式 PRD

## 阅读顺序
1. 先读 `当前产品需求.md`
2. 再读 `模块需求导航.md`
3. 再按当前任务进入 `../实现规范/`、`../数值策划/` 或 `../界面交互/`
4. 最后按需查看 `修订日志.md`

## 真源说明
本目录属于产品方向总控层。
- 本目录不再承担所有模块细节说明
- 模块细节统一下沉到 `../实现规范/`
- 数据字段与运行时配置统一下沉到 `../数值策划/`
- 若本目录与局部实现规范冲突，以本目录判断产品方向，以实现规范判断落地约束

## 使用约束
- 不跳过本目录直接按个人理解扩玩法
- 不把实现细节全文重新堆回本目录
- 若正式产品口径发生变化，优先更新本目录，再同步其他目录说明

## 维护规则
满足以下情况时，增量更新本 README：
- 新增或替换了本目录正式入口文件
- 调整了本目录文件职责或读取顺序
- 明确了新的长期有效边界

以下情况不要更新本 README：
- 只是局部代码修复
- 只是阶段性接力变化
- 只是会话过程记录
