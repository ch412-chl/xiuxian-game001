日期：2026-03-19
最近更新：2026-03-19 17:10:00
文档状态：可用于正式开发 / 联调 / 验收
适用对象：策划、程序、UI、测试

## 1. 文档目的
本文档用于定义《修仙版地下城堡》当前版本的正式产品规格，作为开发实现、资源制作、联调验收和后续迭代的统一依据。

本文档目标：
- 明确当前版本必须实现的玩法范围
- 明确各系统之间的资源流、状态流、交互流
- 明确核心数值逻辑与展示逻辑
- 明确哪些内容已实现、哪些内容待实现、哪些内容暂不纳入当前版本
- 提供可执行的验收口径，避免“按理解开发”

## 2. 产品概述
### 2.1 产品定位
一款以“地下城堡1”核心循环为参考、以修仙世界观为题材的微信小游戏。

### 2.2 玩法流程
玩家通过历练副本击败敌人获取修为经验与资源，推进境界成长，再返回洞府查看成就、图鉴与阶段目标，形成主动推进循环。

### 2.3 产品特征
- 修仙题材与地牢推进结合
- 主动探索驱动成长
- 神通按 CD 即时释放，战斗结果受装配顺序影响
- 每个大境界固定为初期 / 中期 / 后期三段成长
- 文本信息、数值反馈与系统资源流组成完整闭环

## 3. 平台与技术边界
- 平台：微信小游戏
- 视角：2D 文字 + 简易 UI + 基础动画
- 操作方式：点击/触摸
- 存档：本地存档（`wx.setStorageSync`）
- 联网需求：无强联网依赖
- 当前版本不包含：排行榜、公会、PVP、多人协作

## 3.1 文档使用规则
- 本文档中的“必须”表示当前版本强制实现
- 本文档中的“可”表示当前版本允许实现但不作为阻塞项
- 本文档未定义的玩法、字段、页面状态，默认不纳入当前版本
- 当实现结果与本文档冲突时，以本文档最新版本为准

## 4. 设计原则
### 4.1 闭环原则
每个系统都必须回答三个问题：
- 产出什么
- 消耗到哪里
- 反馈给谁

若系统只有展示、没有消耗或没有后续作用，则视为设计不完整。

### 4.2 单屏原则
移动端单屏优先展示当前最重要信息，不允许把无效说明、重复数值、零库存内容堆满屏幕。

### 4.3 低认知成本原则
- 外层信息只展示玩家当前决策所需内容
- 复杂数值留在底层结算，不强行前台展示
- 弹框只承载主界面未展示的信息

### 4.4 成长节奏原则
- 前期成长要快，保证短时间内有反馈
- 中期成长逐步拉长，鼓励玩家使用历练、神通成长与资源管理系统
- 高境界成长必须依赖主动推进，而不是挂机等待

## 6. 核心循环
### 6.1 总循环
1. 人物页查看境界、修为与突破条件
2. 历练进入副本探索
3. 遇敌战斗、击败敌人后即时获得修为经验与临时战利品
4. 撤离或通关后战利品结算入纳戒
5. 使用道具、推进突破、解锁更高副本
6. 进入洞府查看成就、图鉴与阶段目标
7. 返回历练继续主动推进

### 6.2 系统资源流
#### 历练
- 消耗：时间、续航、回灵丹、传送符
- 产出：修为经验、灵石、材料、突破材料、装备、神通使用次数

#### 洞府
- 职责：功能中枢
- 包含：成就入口、图鉴入口、阶段信息
- 不提供：挂机修为、挂机灵石、材料产出、炼丹功能

#### 储物袋
- 存放：本次历练中的临时战利品、可带入探索流程的回灵丹/传送符
- 结算：通关/撤离后写入纳戒；失败则丢失战利品

#### 纳戒
- 存放：永久库存
- 来源：历练结算、成就奖励
- 去向：人物使用、突破消耗

### 6.3 页面入口与出口
| 页面 | 入口 | 退出 |
| --- | --- | --- |
| 人物 | 应用启动、底部导航 | 历练、洞府、纳戒 |
| 历练列表 | 底部导航 | 人物、洞府、探索 |
| 探索 | 点击关卡行 | 历练列表、储物袋、怪物详情 |
| 储物袋 | 探索底部按钮 | 探索 |

| 成就 | 洞府页入口 | 洞府 |
| 妖物图鉴 | 洞府页入口 | 洞府 |

## 7. 术语定义
- 大境界：炼气、筑基、结丹等
- 小境界：某大境界中的初期 / 中期 / 后期
- 小境界：每个大境界中的初期 / 中期 / 后期
- 修为经验：用于推进当前小境界进度的经验值
- 破境：从当前大境界后期满修为进入下一大境界初期
- 储物袋：一次历练中的临时库存页
- 纳戒：角色永久库存页
- 神通：战斗技能，按 CD 自动释放

## 8. 角色系统
### 8.1 外层展示属性
外层只展示以下属性：
- 气血

以下属性只保留底层结算，不对外展示：
- 攻击
- 防御
- 神识
- 速度

### 8.2 气血规则
- 气血为生命值，归零则战斗失败
- 气血展示统一为“单值”
- 不展示 `当前/上限`
- 气血数字显示在血条内部
- 敌我血条表现样式必须一致

### 8.3 底层派生属性
底层保留：
- `totalAtk`
- `totalDef`
- `totalSpd`
- `maxHp`

气血上限公式：
`maxHp = 180 + totalDef * 2 + realm.hpBonus`

设计要求：
- 角色进入探索前必须使用统一派生公式重新计算气血上限
- 不允许探索页使用另一套旧公式单独计算气血

### 8.4 展示状态
| 状态 | 展示内容 |
| --- | --- |
| 正常 | 名称、境界、修为、气血、装备、纳戒入口 |
| 可突破 | 正常内容 + 破境入口高亮 |
| 突破冷却 | 正常内容 + 冷却文案 |

## 9. 境界与修为系统
### 9.1 境界结构
大境界列表：
- 炼气
- 筑基
- 结丹
- 元婴
- 化神
- 炼虚
- 合体
- 大乘
- 渡劫
- 真仙
- 金仙
- 太乙
- 大罗
- 道祖

每个大境界仅包含三个小境界：
- 初期
- 中期
- 后期

### 9.2 当前展示格式
统一为：
- `炼气初期`
- `炼气中期`
- `结丹后期`

### 9.3 修为规则
- 修为改为类似经验值的成长资源
- 修为经验只通过关卡内击败敌人获得
- 普通战斗、事件文本、撤离结算、通关结算都不额外发放修为经验
- 当前小境界修为达到 100% 后，若该阶段不是后期，则自动进入下一小境界并清空当前修为经验
- 当前小境界为后期且修为达到 100% 时，进入可突破状态
- 突破成功后进入下一大境界初期，并清空当前修为经验

### 9.4 角色页展示要求
角色页必须展示：
- 当前境界
- 当前修为经验进度条
- 当前修为经验值 / 升级所需经验值
- 是否可突破

### 9.5 修为来源
- 仅限关卡内击败敌人时即时获得
- 经验在敌人死亡时立即写入角色成长
- 已获得经验不因撤离或死亡回退

### 9.6 修为经验公式
公式约束：
- 当前小境界升级需求 = `RealmStageXpNeed[realmStageIdx]`
- 当前进度百分比 = `xp / 当前小境界升级需求`
- 当前进度百分比上限为 100

正式公式：
- `stageXpNeed = RealmStageXpNeed[realmStageIdx]`
- `xpPct = floor(xp / stageXpNeed * 100)`

数值规则：
- `xp` 保留整数累积
- `xpPct` 最大为 100
- 初期 / 中期满经验时自动升级到下一小境界
- 后期满经验时停止继续累积，等待突破

### 9.7 关卡经验规则
- 每个副本内 8 个普通战斗层、1 个小 Boss、1 个大 Boss 都提供固定修为经验
- 当前副本完整通关一次，理论上可使角色从当前小境界升至下一小境界或进入后期突破状态
- 普通怪经验约占单关总经验 38% ~ 42%
- 小 Boss 经验约占单关总经验 16% ~ 18%
- 大 Boss 经验约占单关总经验 40% ~ 46%
- 关卡经验与境界升级需求统一维护在 `product/数值策划/关卡修为经验与境界升级表.csv`
- 敌人单体经验维护在 `product/数值策划/敌人配置总表.csv`

### 9.8 突破规则
- 只有当前大境界后期且修为 100% 时可突破
- 突破必须消耗对应突破丹
- 无突破丹时不可触发突破判定
- 突破成功率：基础 10% + 突破丹 50%
- 每失败一次，基础成功率 +5%，上限 50%
- 总成功率上限 100%
- 失败进入 1 小时冷却
- 可通过广告跳过冷却

突破概率公式：
- `baseBreakRate = 0.1 + breakFailCount * 0.05`
- `baseBreakRateMax = 0.5`
- `danBonus = 0.5`
- `finalBreakRate = min(min(baseBreakRate, baseBreakRateMax) + danBonus, 1)`

### 9.9 突破后结果
突破成功后：
- 境界进入下一大境界初期
- 当前修为经验清零
- 重新计算气血上限
- 触发渡劫特效
- 返回成功提示文案

洞府为功能中枢页，不承担挂机产出、资源生产或成长加速职责。

- 承接成就入口
- 承接妖物图鉴入口

### 10.3 当前版本不提供

- 离线收益
- 灵石产出加成
- 灵矿
- 灵仆

### 10.4 页面结构

- 顶部标题区
- 中部功能入口区
- 底部主导航

中部功能入口区至少包含：

## 11. 道具系统
### 11.1 当前版本分类

### 11.4 当前版本不提供
- 洞府炼丹
- 丹方生产链
- 聚灵丹

## 12. 纳戒与储物袋
### 12.1 纳戒
定义：角色永久库存页面。

包含：
- 丹药
- 材料
- 装备
- 结算后的战利品

规则：
- 0 数量物品不展示
- 丹药可点击使用或查看
- 装备可点击穿戴/卸下
- 材料仅展示，不可直接使用

### 12.2 储物袋
定义：一次历练中的临时库存页。

包含：
- 可在本次探索内使用的回灵丹
- 可在本次探索内使用的传送符
- 当前关卡灵石
- 当前关卡材料战利品

规则：
- 回灵丹 / 传送符展示数量 = 纳戒可用数量 + 本次历练中途掉落数量
- 探索页本体不直接展示储物袋详细内容
- 点击“储物袋”进入独立子页面
- 0 数量项不展示
- 仅允许使用回灵丹和传送符
- 通关或成功撤离后战利品写入纳戒
- 失败或认命离开则战利品丢失

### 12.3 页面差异
| 项目 | 纳戒 | 储物袋 |
| --- | --- | --- |
| 库存生命周期 | 永久 | 单次历练 |
| 展示来源 | 历练结算、成就奖励 | 当前历练中途 |
| 可使用物品 | 丹药、装备 | 回灵丹、传送符 |
| 主导航 | 隐藏 | 隐藏 |

## 14. 神通系统
### 14.1 基本规则
- 每个大境界仅配置 1 个神通
- 最多装配 3 个神通
- 神通在战斗中按 CD 自动释放
- 同等就绪条件下按装配顺序优先
- 已装配神通只剩 1 个时不允许移除

### 14.2 神通成长阶段
- 入门
- 小成
- 大成
- 圆满
- 超越
- 仙道

### 14.3 升级规则
通过对敌使用次数累计升级：
- 入门 → 小成：20 次
- 小成 → 大成：40 次
- 大成 → 圆满：60 次
- 圆满 → 超越：80 次
- 超越 → 仙道：100 次
- 累计满级：300 次

### 14.4 神通效果规则
- 神通伤害直接展示数值，不展示倍率
- 特殊效果只保留“回复气血”
- 其他增益、减益、属性修饰类描述不对外展示

### 14.5 神通详情弹框
只显示：
- 境界 · 神通名称
- 等级
- 伤害
- 冷却
- 特殊效果（仅当存在回复气血时显示）

不显示：
- 碎片数量
- 倍率
- 冗长描述文案

### 14.6 神通列表规则
- 历练页下方神通区按境界分组展示
- 默认只展示已解锁神通与相邻未解锁神通
- 未解锁神通显示为入门，可查看但不可装配
- 单屏优先展示 3 个已装配槽与当前可选神通列表
- 神通数量不足一屏时不启用滚动
- 神通数量超出一屏时启用纵向滚动

### 14.7 神通页交互优化
交互目标：

- 让“出手顺序”优先于“收藏展示”
- 避免 6 槽时代的大面积空位占屏

交互规则：

- 待替换状态下，神通列表只高亮可装配项
- 点击神通卡后直接替换到当前槽位，不再强依赖二次弹框确认
- 若该神通当前已装配在其他顺位，则点击后直接完成顺位重排
- 点击已装配神通卡可查看详情；详情弹框内允许移除，但当仅剩 1 个已装配神通时移除按钮置灰
- 历练页主界面不再追求“展示全部神通收藏”，而是优先服务装配决策
- 当玩家切换境界时，若该境界只有 1 个神通，则列表中只保留该神通卡与未解锁占位卡
- 因每境界仅 1 神通，神通区视觉重点应从“数量管理”改为“顺序管理”

## 16. 战斗系统
### 16.1 战斗形式
- 即时战斗
- 按行动 CD + 技能 CD 驱动
- 非回合制

### 16.2 战斗参与对象
- 玩家
- 单一敌人

当前版本采用单目标结算。

### 16.3 战斗规则
- 玩家自动按装配顺序释放已就绪神通
- 敌人自动按技能顺序释放已就绪技能
- 支持命中 / 闪避 / 暴击
- 战斗不直接恢复气血

### 16.3.1 战斗结算公式
玩家神通 CD：
- `playerSkillCd = max(0.9, baseCd * max(0.82, 1 - (stLv - 1) * 0.03) * max(0.75, 1.2 - spd / 400))`

角色行动 CD：
- `actorActionCd = max(0.5, 1.35 * (100 / (100 + spd * 0.6)))`

玩家伤害：
- `skillBaseDamage = floor(skillDmg * stPowerMul)`
- `skillRandomDamage = floor(skillBaseDamage * random(0.92, 1.08))`
- `finalDamage = max(1, skillRandomDamage)`

敌方伤害：
- `enemyRandomDamage = floor(enemyBaseDamage * random(0.92, 1.08))`
- `finalEnemyDamage = max(1, enemyRandomDamage)`

治疗神通：
- `healAmount = floor(playerMaxHp * healPct * stPowerMul)`

约束：
- 所有最终伤害至少为 1
- 所有最终治疗至少为 1
- 当前版本群体神通统一按单体结算

### 16.3.2 战斗结果输出
战斗阶段允许对玩家可见的结果只有：
- 血条变化
- 血条数字变化
- CD 条变化
- 怪物详情弹框信息

战斗阶段不对外显示：
- 战斗日志列表
- 属性加成说明
- 额外战斗旁白

### 16.4 视觉表现规范
必须具备：
- 敌方血条变化
- 敌方血条拖尾
- 敌方血条受击抖动
- 我方血条稳定更新
- 技能格 CD 条平滑变化

不允许出现：
- 大段战斗日志文本
- 多余感叹号提示
- 未释放技能前提前展示空 CD 区

### 16.5 按钮规范
未遇敌时底部按钮：
- 自动推进
- 探索
- 储物袋

遇敌时底部按钮：
- 撤离
- 战斗
- 储物袋

顺序不可颠倒。

战斗中：
- 隐藏撤离按钮
- 隐藏战斗按钮
- 隐藏储物袋按钮
- 保留我方血条、神通格与敌方详情入口

### 16.6 战斗显示刷新规则
- 血条变化优先于 CD 条变化
- 受击抖动与拖尾在伤害结算后立即触发
- 战斗结束后立即隐藏 CD 区
- 战斗结束后保留敌方信息，直到进入下一状态

## 20. 数值与平衡要求
### 20.1 当前版本重点平衡项
- 回灵丹续航能力
- 突破成功率与冷却成本
- 历练击杀经验节奏
- 敌方伤害与神通伤害的时间曲线
- 3 槽神通装配下的出手顺序收益

### 20.1.1 数值维护文件
- `product/数值策划/关卡修为经验与境界升级表.csv` 为当前版本的境界成长与升级经验真源。
- `product/数值策划/敌人配置总表.csv` 为当前版本的敌人强度、掉落与修为奖励真源。
- `product/分析报告/关卡修为经验分析.md` 为当前版本的经验节奏说明文件，用于解释主表口径，不作为独立真源。
- 当说明文件与当前实现不一致时，以本章节“当前正式数值表”、上述两份主表与代码实现为当前正式口径。
- 数值修改流程必须遵循：
  1. 先修改 `关卡修为经验与境界升级表.csv` 或 `敌人配置总表.csv`
  2. 再确认分析导出结果是否符合目标节奏
  3. 再同步 PRD 正式公式与代码实现

### 20.1.2 当前正式数值表
| 项目 | 公式/数值 |
| --- | --- |
| 回灵丹效果 | 当前最大气血 `+30%` |

### 20.1.3 数值校验项
- 前期探索一场战斗需至少能看清 1 次血条变化与 1 次 CD 变化
- 炼气阶段玩家不依赖丹药也可完成前置推进
- 回灵丹可提升续航，但不能覆盖全部伤害压力
- 当前副本完整通关一次应能稳定推进到下一小境界或后期突破状态
- 3 槽装配下，玩家必须能在当前境界、中一境、低一境之间完成明确的出手顺序取舍
- 当前版本的玩家与敌人门槛分析不再依赖煞气字段

### 20.2 数值设计原则
- 前期成长时间由历练主动推进决定，不由挂机决定
- 战斗信息必须保证玩家可识别血条变化和 CD 变化
- 中期成长由历练、突破、道具与战斗资源共同构成
- 战斗、成长、经济三条公式链必须在同一维护文件中可追溯，禁止只改展示文案不改源公式

### 20.3 下一版正式建议数值方案
- 当前活动目录下不再默认维护独立的“下一版数值建议文件”。
- 若后续需要重启下一版数值提案，必须在 `product/数值策划/` 中新建正式文件，并同步写入修订日志。
- 已归档的旧成长目标表、旧难度分析表仅供历史追溯，不代表本版本已经正式生效。

## 21. 异常与边界处理
### 21.1 页面展示边界
- 0 数量条目不展示
- 弹框点击空白关闭
- 过长文本必须换行或截断
- 小屏不能出现文字贴边、遮挡、重叠

### 21.2 交互边界
- 仅剩 1 个装配神通时不允许移除
- 无突破丹时不允许突破
- 无传送符时撤离需明确二次确认

### 21.3 战斗边界
- 未发生技能释放前不显示技能 CD 区
- 怪物未遭遇前不显示敌方信息
- 角色进入副本时气血必须校正为满血

### 21.4 页面空态
| 页面 | 空态文案 |
| --- | --- |
| 纳戒 | 暂无物品 |
| 储物袋 | 暂无额外战利品 |
| 成就列表 | 当前无可领取奖励 |

- 当前版本不再保留煞气设定，人物、敌人、门槛分析与战斗结算均不使用煞气
