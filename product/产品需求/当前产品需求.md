## 1. 文档定位
本文档是当前版本的总控 PRD，只负责定义：
- 当前版本要做什么
- 当前版本不做什么
- 各系统的全局边界、资源流、存档流、验收口径
- 正式真源文件与实现规范文件的使用关系

## 2. 产品定位
### 2.1 产品基准
- 世界观：修仙
- 战斗骨架：**对标《向僵尸开炮》**
- 最终落点：修仙自动施法生存肉鸽

### 2.2 当前版本目标
交付一条可运行、可体验、可上线的轻肉鸽主链：
- 人物页查看成长、目标与突破条件
- 历练页选择关卡与整备神通
- 进入单局战场后承载 20 波敌潮、20 级、20 次强化的自动施法战斗
- 洞府页承接成就、图鉴与阶段目标
- 纳戒 / 储物袋形成单次历练与永久库存闭环
- 境界成长、突破、广告补救、成就奖励形成最小可运营循环

### 2.3 当前版本不做
- 十重成长
- 煞气系统
- 洞府挂机与生产链
- 完整自由移动的重型生存肉鸽第一版
- 新美术资源批量接入

## 3. 当前版本正式战斗口径
1. 每局固定 **20 波**
2. 每局局内等级上限 **20 级**
3. 每升 1 级固定触发 **1 次三选一强化**，理论最多 **20 次强化**
4. 第 5 / 10 / 15 波为精英波
5. 第 20 波为首领终局
6. 战斗实时骨架必须对齐《向僵尸开炮》：敌潮持续压近、局内高频成长、build 在一局内明显成型

## 4. 全局设计原则
1. 主链优先：一切优化优先服务人物 -> 历练 -> 战场 -> 结算 -> 洞府
2. 真源唯一：文档、代码、主表冲突时必须回到正式真源
3. 低认知成本：外层只展示当前决策必须信息
4. 资源闭环：每个系统都必须回答“产出什么、消耗到哪里、反馈给谁”

## 5. 核心循环
1. 人物页查看当前境界、经验与突破条件
2. 历练页选择当前主推关卡并进入整备
3. 战前整备页确认 3 神通顺序
4. 进入 20 波单局战场，击杀敌潮获得局内经验、升级并三选一强化
5. 成功撤离或通关后，将本次战利品在 `settle_commit` 写入纳戒
6. 返回人物页 / 洞府页查看成长、成就、图鉴与阶段目标
7. 满足条件后破境，进入下一境界主线

## 6. 全局系统规则
### 6.1 角色外显规则
外层只展示：
- 气血
- 境界
- 修为经验进度
- 是否可突破

### 6.2 境界与修为规则
- 大境界共 14 个
- 每个大境界只有 `初期 / 中期 / 后期`
- 修为经验只通过关卡内击败敌人即时获得
- 后期满经验后进入可突破状态

### 6.3 神通规则
- 每个大境界仅 1 个神通
- 最多装配 3 个局外基础神通
- 战斗中按 CD 自动释放
- 局内通过升级获得强化与临时 build 成长

### 6.4 库存规则
- 纳戒是永久库存
- 储物袋是单次历练库存
- 成功撤离 / 通关才允许将本次战利品写回纳戒
- 失败 / 认命离开会丢失本次战利品，但经验保留

### 6.5 成就与图鉴规则
- 成就与图鉴都从洞府进入
- 成就奖励领取后直接进入纳戒
- 妖物图鉴在首次遭遇时解锁，不要求首次击杀

### 6.6 广告规则
当前版本广告仅允许用于：
- 战斗失败后的复活兜底
- 探索中的撤离兜底
- 跳过突破失败冷却
