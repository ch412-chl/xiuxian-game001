# 数值联动维护说明

版本：v0.9.9
日期：2026-03-18

本文件配套维护表：`数值公式总表.xml`

## 维护目标
- 统一维护境界、洞府、战斗、资源四条数值链路。
- 把“基础参数”和“演算结果”分开，避免手工改一处漏三处。
- 作为“当前实现 + 调优演算”的统一维护入口使用。

## 工作簿结构
- `README`：维护说明与使用顺序。
- `Review`：当前设计链路的合理性判断。
- `ProgressParams`：修为/境界基础参数。
- `ProgressCalc`：根据参数自动推导各境界分钟、XP、加成。
- `CaveParams`：洞府基础参数。
- `CaveCalc`：根据参数自动推导各级洞府产出、升级目标时长、升级成本。
- `CombatSandbox`：可手改输入值，立即看到伤害、减伤、煞气的演算结果。
- `ResourceRules`：炼丹、突破、煞气等资源规则维护表。

## 使用方式
1. 优先修改 `ProgressParams`、`CaveParams`、`CombatSandbox` 输入区。
2. 查看 `ProgressCalc`、`CaveCalc` 是否得到预期结果。
3. 若为正式生效数值，再同步 PRD 与代码。

## 结论
- 当前 PRD 的挂机主线节奏基本成立，但战斗与经济联动存在断链。
- 最大问题不是单个数字，而是三条链路没有闭合：
  - 攻击成长 -> 战斗输出
  - 洞府产出 -> 洞府升级
  - 炼丹/突破 -> 时间成本
